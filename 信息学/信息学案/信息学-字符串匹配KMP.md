# 信息学专题：字符串匹配 KMP

> 信息学 | 字符串算法 | 失配跳转

---

## 本课学什么

KMP 解决的是：在文本串里找模式串出现的位置时，怎样避免“已经比过的字符再白白重比一遍”。

本课重点掌握：

- `next` / `fail` 数组表示什么；
- 匹配失败时为什么可以直接跳；
- KMP 为什么能做到线性复杂度。

---

## 核心知识 / 核心方法

### 1. KMP 的直觉

朴素匹配失败时，模式串常会回退很多，造成重复比较。

KMP 的关键是：**利用已经匹配成功的前缀信息，决定模式串下一步应该跳到哪里。**

### 2. `next` 数组的含义

`next[i]` 常表示：模式串前 `i` 个字符中，最长相等前后缀的长度。

换句话说，当匹配到这里失败时，模式串可以退回到这个长度继续比。

### 3. 求 `next`

```cpp
vector<int> getNext(const string& p) {
    int m = p.size();
    vector<int> ne(m, 0);
    for (int i = 1, j = 0; i < m; ++i) {
        while (j > 0 && p[i] != p[j]) j = ne[j - 1];
        if (p[i] == p[j]) ++j;
        ne[i] = j;
    }
    return ne;
}
```

### 4. KMP 匹配

```cpp
int kmp(const string& s, const string& p) {
    auto ne = getNext(p);
    for (int i = 0, j = 0; i < (int)s.size(); ++i) {
        while (j > 0 && s[i] != p[j]) j = ne[j - 1];
        if (s[i] == p[j]) ++j;
        if (j == (int)p.size()) return i - j + 1;
    }
    return -1;
}
```

### 5. 为什么是线性复杂度

文本串指针 `i` 只向前走；模式串指针 `j` 虽然会回退，但总回退次数有限，因此总复杂度是 `O(n + m)`。

---

## 易错点 / 关键提醒

1. **背代码，不理解 `next` 的意义**。
2. **下标体系混乱**，`0` 下标和 `1` 下标写法不要混。
3. **匹配成功后不会继续找下一个出现位置**。
4. **把 KMP 当成所有字符串题的唯一解**，有时 Hash、Trie、后缀结构更合适。

## 练习题

1. `next` 数组记录的核心信息是什么？
2. 为什么 KMP 比朴素匹配更快？
3. 匹配到一个完整模式串后，若要继续找后续出现位置，应怎样处理 `j`？

## 答案

1. 记录可复用的最长相等前后缀长度。
2. 因为失配后不会让文本串回退，避免了重复比较。
3. 令 `j = ne[j - 1]`，保留可继续复用的前缀信息。
