## 核心代码片段

### 1. 矩阵乘法

适用场景：线性递推、状态转移、图上的固定步数转移。

```cpp
vector<vector<long long>> matMul(
    const vector<vector<long long>>& A,
    const vector<vector<long long>>& B,
    long long mod
) {
    int n = A.size();
    int m = B[0].size();
    int p = B.size();
    vector<vector<long long>> C(n, vector<long long>(m, 0));
    for (int i = 0; i < n; ++i) {
        for (int k = 0; k < p; ++k) {
            for (int j = 0; j < m; ++j) {
                C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % mod;
            }
        }
    }
    return C;
}
```

关键提醒：

- 要先确认矩阵维度可乘；
- 模板中的 `n` 不一定总是方阵，最好写成通用版本。

### 2. 矩阵快速幂

适用场景：递推式形如“下一状态由当前状态线性转移得到”，且指数很大。

```cpp
vector<vector<long long>> matPow(vector<vector<long long>> a, long long b, long long mod) {
    int n = a.size();
    vector<vector<long long>> res(n, vector<long long>(n, 0));
    for (int i = 0; i < n; ++i) res[i][i] = 1;
    while (b > 0) {
        if (b & 1) res = matMul(res, a, mod);
        a = matMul(a, a, mod);
        b >>= 1;
    }
    return res;
}
```

### 3. 组合数取模

适用场景：多次查询组合数，且模数支持逆元处理。

```cpp
long long C(int n, int m, long long mod,
            const vector<long long>& fac,
            const vector<long long>& invfac) {
    if (m < 0 || m > n) return 0;
    return fac[n] * invfac[m] % mod * invfac[n - m] % mod;
}
```

关键提醒：

- 使用前要先预处理 `fac` 和 `invfac`；
- 不是所有模数都能直接这样求逆元。

### 4. 埃氏筛

```cpp
vector<int> primes;
vector<bool> isPrime(n + 1, true);
isPrime[0] = isPrime[1] = false;
for (int i = 2; i * i <= n; ++i) {
    if (isPrime[i]) {
        for (int j = i * i; j <= n; j += i) {
            isPrime[j] = false;
        }
    }
}
for (int i = 2; i <= n; ++i) {
    if (isPrime[i]) primes.push_back(i);
}
```

### 5. 最长回文子串（Manacher 框架）

适用场景：在线性时间内求最长回文子串。

```cpp
string longestPalindrome(const string& s) {
    string t = "^";
    for (char c : s) {
        t += '#';
        t += c;
    }
    t += "#$";

    int n = t.size();
    vector<int> p(n, 0);
    int center = 0, right = 0;
    int bestLen = 0, bestCenter = 0;

    for (int i = 1; i < n - 1; ++i) {
        int mirror = 2 * center - i;
        if (i < right) p[i] = min(right - i, p[mirror]);
        while (t[i + 1 + p[i]] == t[i - 1 - p[i]]) ++p[i];
        if (i + p[i] > right) {
            center = i;
            right = i + p[i];
        }
        if (p[i] > bestLen) {
            bestLen = p[i];
            bestCenter = i;
        }
    }

    int start = (bestCenter - bestLen) / 2;
    return s.substr(start, bestLen);
}
```

关键提醒：

- 这类模板最容易因为边界哨兵写错而越界；
- 如果只是中等数据范围，中心扩展法有时更容易手写正确。

### 6. 最长公共前缀

```cpp
string longestCommonPrefix(vector<string>& strs) {
    if (strs.empty()) return "";
    string res;
    for (int i = 0; i < (int)strs[0].size(); ++i) {
        char c = strs[0][i];
        for (int j = 1; j < (int)strs.size(); ++j) {
            if (i >= (int)strs[j].size() || strs[j][i] != c) return res;
        }
        res += c;
    }
    return res;
}
```

### 7. 自定义排序与比较器

#### `set` / `multiset` 自定义比较

```cpp
struct Node {
    int x, y;
    bool operator<(const Node& other) const {
        if (x != other.x) return x < other.x;
        return y < other.y;
    }
};

set<Node> s;
```

关键提醒：

- 比较器必须形成**严格弱序**；
- 写错比较规则会导致容器行为异常。

#### `priority_queue` 自定义比较

```cpp
struct Node {
    int val, pri;
};

struct Cmp {
    bool operator()(const Node& a, const Node& b) const {
        return a.pri > b.pri; // 小根堆：优先级小的先出
    }
};

priority_queue<Node, vector<Node>, Cmp> pq;
```

### 8. 自定义哈希

适用场景：降低某些构造数据下 `unordered_map` 被卡常或冲突严重的风险。

```cpp
struct custom_hash {
    static uint64_t splitmix64(uint64_t x) {
        x += 0x9e3779b97f4a7c15;
        x = (x ^ (x >> 30)) * 0xbf58476d1ce4e5b9;
        x = (x ^ (x >> 27)) * 0x94d049bb133111eb;
        return x ^ (x >> 31);
    }

    size_t operator()(uint64_t x) const {
        static const uint64_t FIXED_RANDOM =
            chrono::steady_clock::now().time_since_epoch().count();
        return splitmix64(x + FIXED_RANDOM);
    }
};

unordered_map<long long, int, custom_hash> mp;
```

```cpp
inline int read() {
    int x = 0, f = 1;
    char c = getchar();
    while (c < '0' || c > '9') {
        if (c == '-') f = -1;
        c = getchar();
```

## 易错点 / 关键提醒

1. **矩阵模板最容易错在维度和取模位置**。  
2. **Manacher 模板最容易错在边界和下标换算**。  
3. **比较器如果不满足严格弱序，`set` 和 `priority_queue` 都可能行为异常**。  
4. **自定义哈希不是必需品，只在哈希冲突风险明显时再考虑**。  
5. **快速读入模板要确认题目输入格式简单，否则混用 `cin` / `getchar` 容易出问题**。

## 例题

### 例1：线性递推很长，应该想到什么

某数列满足固定线性递推，要求第 `10^18` 项模某个数。此时暴力递推显然不行，优先想到什么？

#### 解析

当状态转移可以写成矩阵形式时，矩阵快速幂能把复杂度从线性降到对数级。

#### 答案

**矩阵快速幂**

### 例2：为什么 `unordered_map` 有时要加自定义哈希

#### 解析

因为在某些特殊数据下，默认哈希可能冲突较多，影响性能。  
竞赛中在高强度卡常题里，会考虑自定义哈希来增强稳定性。

#### 答案

**为了降低哈希冲突带来的性能风险。**

### 例3：比较器为什么重要

#### 解析

比较器决定了容器内部如何判定“谁更小”“谁优先”。  
一旦比较规则写错，容器内部顺序就可能失真，甚至产生难以发现的错误。

#### 答案

**因为很多STL容器的行为直接建立在比较规则正确的前提上。**

1. 矩阵快速幂适合解决哪一类问题？  
2. `unordered_map<long long, int, custom_hash>` 中自定义哈希的主要目的是什么？  
3. 为什么 Manacher 模板不建议在没理解下标含义时直接硬背？  
4. `priority_queue` 自定义比较器时，最需要保证什么性质？  
5. 快速读入模板在什么情况下要谨慎使用？

---

## 答案

1. **线性递推或可写成固定状态转移矩阵的问题**。  
2. **降低哈希冲突导致的性能风险**。  
3. **因为它非常依赖边界与中心半径的下标定义，照抄容易越界或还原位置出错**。  
4. **比较规则要正确、稳定，并满足严格弱序**。  
5. **输入格式复杂、需要和 `cin` 混用、或含大量特殊字符时要谨慎**。
