# 考点11：STL容器

> 信息学｜C++工具与算法实现｜重点是“按场景选容器，而不是死记接口”

---

## 本课学什么

STL容器是竞赛与算法题中的常用工具。  
本课不只是记住 `set`、`map`、`queue` 的写法，更重要的是理解：**什么数据应该放进什么容器，为什么这样选，复杂度代价是什么**。

学完后，你应该能够：

- 区分顺序容器、关联容器、无序容器和容器适配器；
- 根据“是否有序、是否允许重复、是否要按键查找”选择合适容器；
- 避开 `erase`、`operator[]`、迭代器失效等常见错误。

---

## 核心知识 / 核心方法

### 1. 先按“需求”选容器

做题时可先问自己四个问题：

1. 元素要不要保持顺序？
2. 元素可不可以重复？
3. 查找是按“位置”还是按“键”？
4. 更看重稳定有序还是平均常数更小？

### 2. 常见容器分类

#### 顺序容器

| 容器 | 特点 | 常见用途 |
| --- | --- | --- |
| `vector` | 连续存储，支持随机访问 | 数组替代、动态开空间 |
| `deque` | 两端操作较快 | 双端队列、滑动窗口 |
| `list` | 链表结构 | 很少在竞赛主流题中使用 |

竞赛里最常用的顺序容器通常是 **`vector`**。

#### 关联容器

底层通常是平衡树，特点是**有序**。

| 容器 | 是否允许重复 | 是否按键有序 |
| --- | --- | --- |
| `set` | 否 | 是 |
| `multiset` | 是 | 是 |
| `map` | key不重复 | 是 |
| `multimap` | key可重复 | 是 |

常见复杂度：插入、删除、查找一般为 **`O(log n)`**。

#### 无序关联容器

底层通常是哈希表，特点是**平均更快，但不保证有序**。

| 容器 | 特点 |
| --- | --- |
| `unordered_set` | 去重、平均 `O(1)` 查找 |
| `unordered_map` | 键值映射、平均 `O(1)` 查找 |

注意：无序容器的遍历顺序**不可当作有序结果使用**。

#### 容器适配器

| 容器 | 本质 |
| --- | --- |
| `stack` | 后进先出 |
| `queue` | 先进先出 |
| `priority_queue` | 默认大根堆 |

它们强调操作方式，不强调遍历。

### 3. 高频容器的核心用法

#### `set`

```cpp
set<int> s;
s.insert(5);
s.insert(3);
if (s.count(5)) {
    // 5存在
}
s.erase(3);
```

适合：

- 去重；
- 维护有序集合；
- 查询前驱、后继、最小值、最大值。

#### `multiset`

```cpp
multiset<int> ms;
ms.insert(2);
ms.insert(2);
auto it = ms.find(2);
if (it != ms.end()) ms.erase(it); // 只删一个
```

关键点：  
`ms.erase(2)` 会把值为 2 的元素**全部删掉**，而 `erase(it)` 只删一个。

#### `map`

```cpp
map<string, int> mp;
mp["apple"] = 3;
mp["banana"]++;
if (mp.count("apple")) {
    cout << mp["apple"] << '\n';
}
```

适合：

- 统计次数；
- 建立“键 → 值”的有序映射；
- 需要按 key 从小到大遍历时。

#### `unordered_map`

```cpp
unordered_map<string, int> cnt;
cnt["cat"]++;
if (cnt.count("dog")) {
    cout << cnt["dog"] << '\n';
}
```

适合：

- 只关心查找速度，不关心顺序；
- 做计数、映射、判重。

#### `priority_queue`

```cpp
priority_queue<int> maxHeap; // 大根堆

priority_queue<int, vector<int>, greater<int>> minHeap; // 小根堆
```

适合：

- 每次都要快速取当前最大/最小元素；
- Dijkstra、贪心、Top K 等问题。

### 4. 常见场景与容器选择

| 场景 | 推荐容器 | 原因 |
| --- | --- | --- |
| 去重并保持有序 | `set` | 自动去重且有序 |
| 元素可重复且要有序 | `multiset` | 支持重复元素 |
| 键值映射且要按key排序 | `map` | 自动按key有序 |
| 只想快速查找键值 | `unordered_map` | 平均查找更快 |
| 维护队首出队、队尾入队 | `queue` | 先进先出 |
| 维护当前最值 | `priority_queue` | 取堆顶快 |
| 可变长数组 | `vector` | 使用最灵活 |

---

## 易错点 / 关键提醒

1. **`unordered_map` 不保证遍历顺序**  
   需要有序结果时不能用它代替 `map`。

2. **`map[key]` 会在 key 不存在时自动创建元素**  
   只想判断是否存在时，优先用 `count` 或 `find`。

3. **`priority_queue` 默认是大根堆**  
   需要最小值优先时，要显式写比较器。

4. **删除 `multiset` 中的一个元素时，要用迭代器删**  
   否则可能把相同值全部删掉。

5. **复杂度要和题目规模对应**  
   `n` 很大时，频繁 `O(log n)` 和平均 `O(1)` 的差距可能会很明显。

## 例题

### 例1：去重并排序

给定一个整数数组，要求输出去重后的升序结果。应优先选择什么容器？

#### 解析

需要同时满足：

- 去重；
- 自动排序。

`set` 正好满足。

#### 参考代码

```cpp
vector<int> a = {4, 2, 4, 1, 3, 2};
set<int> s(a.begin(), a.end());
for (int x : s) cout << x << " ";
```

#### 答案

**选择 `set`。**

### 例2：统计单词出现次数

给定若干单词，要求统计每个单词出现次数。若不要求按字典序输出，优先选什么？

#### 解析

这是典型的“键 → 次数”映射问题，不要求有序时优先选 `unordered_map`。

#### 参考代码

```cpp
unordered_map<string, int> cnt;
for (string s : words) cnt[s]++;
```

#### 答案

**优先选 `unordered_map<string, int>`。**

### 例3：维护动态中位数时为什么常用两个堆

#### 解析

因为 `priority_queue` 可以高效维护一半较小元素中的最大值、另一半较大元素中的最小值。  
这类题的本质不是“排序全部元素”，而是“反复取当前边界值”。

#### 答案

**因为堆适合维护动态最值，能高效支持中位数平衡结构。**

## 练习题

1. 需要“有序去重”时，应该选 `set` 还是 `unordered_set`？为什么？  
2. `priority_queue<int>` 默认取出的是什么元素？  
3. 为什么统计次数时经常使用 `map` 或 `unordered_map`？  
4. 想删除 `multiset` 中的一个 `x`，为什么常先 `find` 再 `erase(it)`？  
5. 如果题目要求按 key 从小到大输出映射结果，应选 `map` 还是 `unordered_map`？

---

## 答案

1. **选 `set`，因为它既能去重，又能保持有序**。  
2. **当前最大元素**。  
3. **因为它们天然支持“键 → 值”的映射结构，适合做计数和索引**。  
4. **因为直接 `erase(x)` 可能删掉所有值为 `x` 的元素，而不是只删一个**。  
5. **选 `map`**。
