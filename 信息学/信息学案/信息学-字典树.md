# 信息学专题：字典树

> 信息学 | 字符串结构 | 前缀检索

---

## 本课学什么

字典树（Trie）适合处理大量字符串的前缀关系。它的重点不是“把字符串存在树里”，而是：**相同前缀只存一次，从而让前缀查询变得高效**。

本课学习：

- Trie 的节点含义；
- 插入、查找、统计前缀的基本写法；
- 为什么 Trie 适合做前缀问题。

---

## 核心知识 / 核心方法

### 1. Trie 的本质

从根节点出发，按字符一层层往下走：

- 路径表示前缀；
- 节点表示某个前缀状态；
- 终止标记表示某个完整单词结束。

### 2. 节点里通常存什么

- `next[c]`：沿字符 `c` 走向哪个子节点；
- `cnt`：有多少单词在这里结束；
- `pass`：有多少单词经过这里。

### 3. 插入字符串

```cpp
void insert(const string& s) {
    int p = 0;
    for (char ch : s) {
        int c = ch - 'a';
        if (!trie[p].next[c]) trie[p].next[c] = ++tot;
        p = trie[p].next[c];
        trie[p].pass++;
    }
    trie[p].cnt++;
}
```

### 4. 查询完整字符串 / 查询前缀

```cpp
int findWord(const string& s) {
    int p = 0;
    for (char ch : s) {
        int c = ch - 'a';
        if (!trie[p].next[c]) return 0;
        p = trie[p].next[c];
    }
    return trie[p].cnt;
}
```

如果要求“有多少单词以某前缀开头”，返回 `pass` 即可。

---

## 易错点 / 关键提醒

1. **把“查到这个前缀”误当成“这个单词存在”**。
2. **忘记区分 `cnt` 和 `pass`**。
3. **字符范围没想清楚**，字母、小写、大写、数字都可能影响数组大小。
4. **节点数估计过小**，容易越界。

## 例题

### 例1：基础例题——单词是否出现过

插入若干单词后，查询某个单词是否存在。

**关键点**：最终应检查结束标记 `cnt`，而不是只看路径是否存在。

### 例2：应用例题——统计前缀个数

例如统计有多少单词以 `ab` 开头。

**做法**：沿 `a -> b` 走到对应节点，读取 `pass`。

---

## 练习题

1. 为什么 Trie 特别适合前缀查询？
2. `cnt` 和 `pass` 分别表示什么？
3. 若字母范围从 `a-z` 扩展到数字和大小写字母，节点结构应如何调整？

## 答案

1. 因为相同前缀共用路径，查前缀只需按字符往下走。
2. `cnt` 表示在该节点结尾的单词数，`pass` 表示经过该节点的单词数。
3. 扩大字符映射范围，或改用哈希表存孩子指针。
