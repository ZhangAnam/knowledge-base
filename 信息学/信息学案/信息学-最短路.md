# 信息学专题：最短路

> 信息学 | 图论 | 路径优化

---

## 本课学什么

最短路问题研究的是：在图上从一个点走到另一个点，总代价最小的路径是多少。

本课重点掌握：

- 单源最短路和多源最短路的区别；
- Dijkstra、Bellman-Ford、Floyd 各自适用什么场景；
- 为什么边权条件会影响算法选择。

---

## 核心知识 / 核心方法

### 1. 问题分类

- **单源最短路**：从一个起点到其他所有点；
- **多源 / 全源最短路**：任意两点之间的最短路；
- **是否允许负边权**：这会直接决定算法能不能用。

### 2. Dijkstra

适用：**边权非负** 的单源最短路。

核心思想：每次确定当前距离最小的未确定点，再用它去更新邻居。

```cpp
void dijkstra(int s) {
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
    dist[s] = 0;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d != dist[u]) continue;
        for (auto [v, w] : adj[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}
```

### 3. Bellman-Ford

适用：允许负边权，并可检测负环。

思想是重复做边松弛。

### 4. Floyd

适用：点数较小的任意两点最短路。

```cpp
for (int k = 1; k <= n; ++k)
    for (int i = 1; i <= n; ++i)
        for (int j = 1; j <= n; ++j)
            dis[i][j] = min(dis[i][j], dis[i][k] + dis[k][j]);
```

### 5. 选型思路

- 非负边 + 单源：优先 Dijkstra；
- 有负边：考虑 Bellman-Ford / SPFA 思想；
- 全源最短路且点数不大：考虑 Floyd。

---

## 易错点 / 关键提醒

1. **有负边还写 Dijkstra**。
2. **只会背模板，不知道 `dist[v] > dist[u] + w` 的含义是松弛**。
3. **图是无向图时漏加反向边**。
4. **初始化无穷大不合理，导致溢出或比较错误**。

---

## 例题

### 例1：基础例题——单源最短路

给定起点 `1`，求到其他所有点的最短距离。若边权均非负，用 Dijkstra。

### 例2：提高例题——任意两点最短路

若点数较小且多次查询任意两点间距离，可用 Floyd 预处理。

---

## 练习题

1. 为什么 Dijkstra 不能直接处理负边？
2. Floyd 的三层循环分别表示什么？
3. 单源非负边最短路常优先选哪种算法？

## 答案

1. 因为“当前最短点已确定”这一性质在负边下可能被破坏。
2. 枚举中转点、起点、终点。
3. Dijkstra（通常是堆优化版）。
