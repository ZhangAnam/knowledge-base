# 信息学5：数组

> 信息学｜基础数据结构

## 本课学什么

当题目需要同时处理很多个同类数据时，数组比单个变量更合适。本课重点是理解“同类型、连续编号、批量处理”这三个关键词。

## 学习目标

- 理解数组是“同类数据按编号存放”的结构。
- 会使用下标读取、修改和遍历数组。
- 能用数组解决批量统计、查找和简单模拟问题。

## 概念

### 1. 数组
数组把许多同类型的数据按顺序存放在一起，每个位置都有下标。它适合处理“第 i 个数”这类问题。

### 2. 下标
下标是访问数组元素的依据。C++ 常用 0 到 n-1，也有题目采用 1 到 n，关键是全程统一。

### 3. 遍历
遍历就是按顺序访问数组中的每个元素。输入、输出、统计、查找通常都建立在遍历上。

## 算法思想 / 分析思路

- 一旦题目出现“n 个数”，就要想到是否需要数组保存。
- 数组题通常分三步：存数据、按下标遍历、更新答案。
- 复杂题往往不是“数组本身难”，而是对下标和边界不够敏感。

## 实现步骤

1. 根据数据规模定义合适大小的数组。
2. 按下标读入所有元素。
3. 通过循环遍历数组完成统计或修改。
4. 若涉及相邻元素，重点检查 i-1、i+1 是否越界。

## 常见错误

1. **下标越界**：访问不存在的位置，是数组题最常见问题。
2. **0 下标与 1 下标混用**：读入和处理阶段若不统一，很容易错位。
3. **数组大小开得不够**：当 n 接近上限时会发生越界。
4. **把位置和值混淆**：a[i] 是值，i 才是位置。

## 例题

### 例1：求数组最大值
**层次**：基础例题

**题目**：输入 n 个整数，输出其中最大值。

**分析**：先把第一个元素当作当前最大值，再逐个比较。

**答案/结论**：数组遍历常与“维护最优答案”配合出现。

### 例2：统计及格人数
**层次**：关键例题

**题目**：输入若干分数，统计及格人数与不及格人数。

**分析**：每个分数都要逐一处理；若后续还要多次使用，保存到数组更灵活。

**答案/结论**：数组适合处理一批同类数据。

### 例3：相邻元素差值
**层次**：易错例题

**题目**：给定数组，输出每对相邻元素的差。

**分析**：这类题需要访问 a[i] 和 a[i-1]，循环通常从 i=1 开始。

**答案/结论**：遇到相邻元素时，先想下标起点。

## 练习

1. 输入 n 个整数，求它们的平均值。
2. 输入数组，输出逆序结果。
3. 统计数组中等于 x 的元素个数。
4. 找出最小值及其第一次出现的位置。

## 答案

1. 平均值可先求和，再除以 n。
2. 逆序输出时下标从最后一个位置往前走。
3. 遍历时遇到等于 x 的元素就计数。
4. 需要同时维护最小值和位置两个信息。
