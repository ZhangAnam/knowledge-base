# 信息学13：二叉树基础

> 信息学｜树形数据结构

## 本课学什么

二叉树是最基础的树结构之一。你需要先看懂“根、左右孩子、叶子、子树”这些关系，后续的遍历、堆、搜索树等内容才容易建立起来。

## 学习目标

- 理解二叉树的基本术语与层级关系。
- 知道节点数、高度、叶子等常见性质。
- 能用“根 + 左子树 + 右子树”的视角分析问题。

## 概念

### 1. 二叉树
二叉树中每个节点最多只有两个孩子，通常称为左孩子和右孩子。左右顺序是有意义的。

### 2. 基本术语
根节点是最上面的起点；叶子节点没有孩子；子树是某个节点及其后代构成的部分结构。

### 3. 常见性质
二叉树常用高度、深度、节点数、叶子数来描述结构。很多性质都与层次有关。

## 算法思想 / 分析思路

- 看树题时先画结构图，再谈代码。
- 二叉树问题常能分解成“根的任务 + 左子树 + 右子树”。
- 遇到节点数、高度、叶子数等统计问题时，递归通常很自然。

## 实现步骤

1. 先找根节点。
2. 分清每个节点的左孩子和右孩子。
3. 判断哪些节点是叶子。
4. 再分析节点总数、高度或其他性质。

## 常见错误

1. **把左右孩子当成无区别的两个孩子**：在二叉树中，左右顺序本身有意义。
2. **深度和高度混淆**：要看题目定义是从根往下算，还是从节点往下算。
3. **叶子判断错误**：左右孩子都不存在才是叶子。
4. **把一般树和二叉树性质混用**：二叉树每个节点至多两个孩子。

## 例题

### 例1：求二叉树节点总数
**层次**：基础例题

**题目**：给定一棵二叉树，求节点总数。

**分析**：每棵树都可以拆成：根节点 + 左子树节点数 + 右子树节点数。

**答案/结论**：这类题天然适合递归。

### 例2：求二叉树高度
**层次**：关键例题

**题目**：给定一棵二叉树，求它的高度。

**分析**：高度等于左右子树高度的较大值再加 1。

**答案/结论**：“根 + 左右子树”思路再次出现。

### 例3：判断叶子节点
**层次**：易错例题

**题目**：某节点只有左孩子，没有右孩子，它是叶子吗？

**分析**：叶子节点的定义是没有任何孩子，而不是孩子不满两个。

**答案/结论**：不是叶子节点。

## 练习

1. 根据图判断哪些节点是叶子。
2. 求二叉树节点个数与叶子节点个数。
3. 判断一棵树的高度。
4. 思考完全二叉树和普通二叉树的区别。

## 答案

1. 叶子必须左右孩子都不存在。
2. 节点数和叶子数都可用递归统计。
3. 高度通常与最长向下路径有关。
4. 完全二叉树强调按层尽量填满。
