# 信息学1：变量与数据类型

> 信息学｜基础编程入门

## 本课学什么

本课解决“数据怎样放进程序”这个最基础的问题。重点不是背类型名字，而是学会根据题目中的整数、小数、字符和字符串选择合适的存储方式，让输入、计算、输出保持一致。

## 学习目标

- 理解变量是“带名字的存储单元”。
- 会根据数据范围区分 int、long long、double、char、string。
- 能写出简单输入输出并检查类型是否匹配。

## 概念

### 1. 变量
变量是程序中保存数据的名字。变量名要能表达含义，后续调试和阅读代码才会清楚。

### 2. 数据类型
数据类型决定变量能存什么、怎样运算、范围多大。题目中的“人数、次数”常用整数，“平均值、长度”常涉及浮点数。

### 3. 类型与题意
先看数据意义，再选类型。若题目可能出现大整数或小数，类型就必须提前选对，不能等结果错了再补救。

## 算法思想 / 分析思路

- 先把题目拆成“要保存哪些量”。
- 再根据是否有小数、范围是否很大来选类型。
- 输入、运算、输出要围绕同一组变量展开。

## 实现步骤

1. 读题并圈出所有要保存的数据。
2. 判断每个量的类型和范围。
3. 给变量起有意义的名字。
4. 按输入顺序读入，按题意完成计算。
5. 输出前检查是否会丢失精度或发生溢出。

## 常见错误

1. **先用后定义**：变量没有声明就使用，编译器无法判断它的类型。
2. **整型范围不够**：可能达到 10^10 的数据仍然用 int，结果会溢出。
3. **字符和字符串混淆**：char 只存一个字符，string 才能存一串字符。
4. **整型除法误判**：两个 int 相除仍是整数结果，小数部分会被截掉。

## 例题

### 例1：交换两个整数
**层次**：基础例题

**题目**：输入两个整数 a、b，交换后输出。

**分析**：交换的关键是先保存原值，避免一个变量被覆盖后另一个值丢失。

**关键步骤**：
- 设置临时变量 t。
- 先 t=a，再 a=b，最后 b=t。

**答案/结论**：借助第三个变量可以稳定完成交换。

### 例2：计算带小数边长的长方形面积
**层次**：关键例题

**题目**：输入长和宽，边长可能为小数，输出面积。

**分析**：题目里出现小数时，变量类型就不能再用 int。

**答案/结论**：length 和 width 更适合定义为 double。

### 例3：平均分为什么不能直接用整型
**层次**：易错例题

**题目**：输入三门成绩，输出平均分。

**分析**：若 total 和 3 都是 int，表达式 total / 3 会先做整型除法。

```text
double avg = total / 3.0
```

**答案/结论**：至少要让表达式进入浮点运算，才能保留小数部分。

## 练习

1. 输入一个整数 n，原样输出它。
2. 输入两个整数，输出它们的和与积。
3. 输入一个字符和一个单词，分别输出。
4. 输入半径 r，输出圆面积，并说明为什么更适合用 double。

## 答案

1. 重点检查是否先定义变量，再读入输出。
2. 若数据范围较大，可把 int 改成 long long。
3. 单个字符用 char，单词用 string。
4. 圆面积通常含小数，使用 double 更稳妥。
