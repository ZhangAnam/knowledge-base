# 信息学21：动态规划入门

> 信息学｜算法设计方法

## 本课学什么

动态规划解决的是“有重复子问题、能由小问题推大问题”的题。重点不是背公式，而是学会找状态、写转移、定边界和安排计算顺序。

## 学习目标

- 理解 DP 的状态、转移、边界、顺序四个核心问题。
- 能区分求最优值、方案数和可行性的 DP。
- 知道为什么 DP 不是机械套表。

## 概念

### 1. 状态
状态用来表示“在某个阶段，需要保留下来供后续使用的信息”。状态定义清楚，转移才会清楚。

### 2. 状态转移
转移说明当前状态怎样由更小状态得到，本质是在描述“最后一步从哪里来”。

### 3. 边界与顺序
边界由状态定义决定，计算顺序则要保证转移所依赖的状态已经算好。

## 算法思想 / 分析思路

- 写 DP 前先问自己：dp[i] 或 dp[i][j] 到底表示什么。
- 若题目是“最大、最小、最多、最少、方案数、是否可行”，常要考虑 DP。
- 很多转移都能从“最后一步”或“最后一次选择”出发思考。

## 实现步骤

1. 定义状态含义。
2. 找出当前状态依赖哪些更小状态。
3. 设置边界和不可达状态。
4. 确定递推顺序或记忆化搜索方式。
5. 用小样例验证转移是否符合题意。

## 常见错误

1. **状态含义不清**：不知道 dp 数组每项代表什么，转移一定会乱。
2. **边界乱设**：边界不是猜出来的，而是由状态定义决定的。
3. **循环顺序写错**：背包类问题尤其容易因此变成另一种模型。
4. **只背公式，不理解来源**：题面稍有变化就不会做。

## 例题

### 例1：爬楼梯
**层次**：基础例题

**题目**：每次走 1 级或 2 级，求到第 n 级的方法数。

**分析**：到第 i 级的最后一步，要么从 i-1 走 1 级来，要么从 i-2 走 2 级来。

**答案/结论**：这是建立 DP 直觉的经典入门题。

### 例2：0/1 背包为什么要倒序
**层次**：关键例题

**题目**：一维优化的 0/1 背包中，容量为什么要倒序枚举？

**分析**：倒序能保证每件物品只使用一次；若正序，本轮更新结果会再次被本轮使用。

**答案/结论**：循环顺序本身就是模型的一部分。

### 例3：最小值 DP 的初始化
**层次**：易错例题

**题目**：求最小值时，为什么初值通常不能设成 0？

**分析**：若 0 不是合法的起始含义，就会错误影响后续最小值比较。

**答案/结论**：初始化本身就是状态含义的一部分。

## 练习

1. 写 DP 前最关键的四个问题是什么？
2. 为什么说状态定义是 DP 的核心？
3. 0/1 背包一维优化时为什么要倒序枚举容量？
4. 记忆化搜索和递推 DP 的共同点是什么？

## 答案

1. 状态、转移、边界、计算顺序。
2. 因为状态定义决定每个数组元素到底表示什么。
3. 为了保证每件物品只被使用一次。
4. 它们都在解决重复子问题，只是实现顺序不同。
