# 考点2：存储器的层次结构

> 信息学｜计算机体系结构基础｜重点理解“为什么要分层、每层解决什么问题”

---

## 本课学什么

本课学习计算机为什么不能只用一种存储器，以及**寄存器、Cache、主存、外存**之间的关系。  
难点不在于死记层次顺序，而在于理解：**速度、容量、价格三者不能同时做到最好，所以必须分层设计**。

学完后，你应该能够：

- 画出常见的存储层次结构；
- 解释局部性原理为什么支持多级存储；
- 区分 SRAM、DRAM、ROM、Flash 等常见存储类型。

---

## 核心知识

### 1. 为什么需要存储层次

理想的存储器应当同时满足：

- 容量大；
- 速度快；
- 价格低。

但现实中这三点往往互相矛盾：

- 越快的存储器，通常越贵、容量越小；
- 越便宜、容量越大的存储器，通常越慢。

所以计算机采用“**少量高速 + 大量低速**”的多级层次结构。

### 2. 常见存储层次结构

从靠近CPU到远离CPU，通常可以写成：

```text
寄存器
↓
Cache（L1 / L2 / L3）
↓
主存（内存）
↓
辅存 / 外存（SSD、HDD等）
↓
光盘、磁带等长期存储
```

总体规律：

| 层次越往上 | 层次越往下 |
| --- | --- |
| 速度更快 | 速度更慢 |
| 容量更小 | 容量更大 |
| 单位成本更高 | 单位成本更低 |

### 3. 局部性原理

多级存储能发挥作用，关键依赖**程序访问的局部性**。

#### 时间局部性

如果某个数据或指令刚被访问过，那么它在不久后很可能还会再次被访问。

例如：

- 循环变量反复使用；
- 循环体内的指令反复执行。

#### 空间局部性

如果某个存储单元被访问，那么它附近的单元也很可能很快被访问。

例如：

- 顺序遍历数组；
- 顺序执行相邻指令。

结论：  
如果把“最近使用过的数据”和“附近的数据”提前放到更快的存储层，就能大幅提高平均访问速度。

### 4. Cache的作用

Cache位于CPU和主存之间，用来缓冲近期最常访问的数据和指令。

常见概念：

- **命中（hit）**：CPU访问的数据就在Cache中；
- **未命中（miss）**：Cache中没有，需要到下一层去取；
- **命中率**：访问中命中的比例。

命中率越高，平均访问时间越短。

### 5. 主存与常见存储器类型

#### SRAM

- 速度快；
- 成本高；
- 常用于 Cache。

#### DRAM

- 容量大、成本较低；
- 速度比SRAM慢；
- 常用于主存。

#### ROM

- 非易失性；
- 适合存放固件、启动程序等不常改动内容。

#### Flash

- 非易失性，可反复擦写；
- 常见于U盘、SSD、存储卡等。

### 6. 逻辑地址与物理地址

程序中使用的地址通常是**逻辑地址（虚拟地址）**，真正送到硬件上的地址是**物理地址**。

引入地址变换的好处包括：

- 让程序不必关心自己被装入内存的哪个具体位置；
- 支持更灵活的内存管理与保护。

## 易错点 / 关键提醒

1. **Cache不是主存，也不是寄存器**  
   它处在两者之间，主要作用是缓冲热点数据。

2. **层次越高，不是容量越大，而是越小越快**。

3. **局部性原理不是硬件结构，而是程序访问特性**  
   正因为程序有局部性，硬件分层才有效。

4. **SRAM通常用于Cache，DRAM通常用于主存**  
   不要把两者用途反过来。

5. **SSD属于外存，不属于主存**  
   即使它速度很快，也不能把它和内存混为一谈。

## 例题

### 例1：判断层次顺序

以下从快到慢排列正确的是哪一项？

A. 主存 → Cache → 寄存器  
B. 寄存器 → Cache → 主存  
C. Cache → 主存 → 寄存器  
D. SSD → Cache → 主存

#### 解析

寄存器离CPU最近、速度最快；然后是Cache；再往下是主存。

#### 答案

**B**

### 例2：解释局部性

程序顺序遍历一个数组，为什么这类访问容易产生空间局部性？

#### 解析

因为数组元素在内存中通常连续存放，访问一个元素后，其相邻元素很可能马上被访问。

#### 答案

**因为被访问位置附近的数据很快也会被访问。**

### 例3：判断存储类型用途

主存通常使用SRAM还是DRAM？为什么？

#### 解析

主存更看重容量和成本平衡，而不是极致速度，因此通常采用DRAM。

#### 答案

**通常使用DRAM，因为它容量更大、成本更低。**

### 例4：Cache存在的意义

如果所有数据最终都在主存中，为什么还需要Cache？

#### 解析

因为CPU访问速度远高于主存，如果每次都直接访问主存，会造成大量等待。  
Cache利用局部性，把热点数据提前放在更快的层级中，减少平均访问延迟。

#### 答案

**为了利用局部性降低平均访问时间。**

## 练习题

1. 存储层次结构为什么不能只用一种存储器？  
2. 什么是时间局部性？请举一个简单场景。  
3. SRAM和DRAM分别更常用于哪一层？  
4. “命中率”描述的是哪一种现象？  
5. SSD属于主存还是外存？

---

## 答案

1. **因为速度、容量、成本三者难以同时最优，只能分层折中**。  
2. **刚访问过的数据很快还可能再次访问，例如循环变量反复使用**。  
3. **SRAM常用于Cache，DRAM常用于主存**。  
4. **描述数据访问时落在Cache中的比例**。  
5. **属于外存**。
