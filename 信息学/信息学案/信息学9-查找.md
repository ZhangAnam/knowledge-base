# 信息学9：查找

> 信息学｜基础算法

## 本课学什么

查找的任务是“在数据中快速找到目标，或判断目标是否存在”。从顺序查找到二分查找，本课要建立“数据是否有序决定策略”的意识。

## 学习目标

- 会区分顺序查找与二分查找。
- 理解二分查找依赖“有序”这一前提。
- 能用查找解决存在性判断、位置定位和边界搜索问题。

## 概念

### 1. 顺序查找
从头到尾逐个比较目标，优点是简单，适合数据量小或数据无序的场景。

### 2. 二分查找
在有序区间中反复取中点，把搜索范围缩小一半。它体现的是“利用有序性减少无效比较”。

### 3. 边界查找
很多题不是找某个具体值，而是找“第一个满足条件的位置”或“最后一个满足条件的位置”。

## 算法思想 / 分析思路

- 先判断数据是否有序；无序时通常先顺序查找，或先排序再二分。
- 二分的核心不是背模板，而是每次都能明确丢掉哪一半。
- 边界查找题要先定义“满足条件”的含义，再设计左右收缩方式。

## 实现步骤

1. 明确目标是找值、找位置还是找边界。
2. 若使用二分，先保证序列有序。
3. 设置 left、right 和 mid。
4. 根据 mid 位置是否满足条件来缩小区间。
5. 循环结束后检查答案是否合法。

## 常见错误

1. **未排序就直接二分**：二分查找最关键的前提就是有序。
2. **区间不收缩**：mid 更新不当会导致死循环。
3. **把找值与找边界模板混用**：条件不同，更新方式也不同。
4. **忘记处理目标不存在**：查找结束后仍要验证结果。

## 例题

### 例1：顺序查找指定数字
**层次**：基础例题

**题目**：在 n 个无序整数中查找 x，输出是否存在。

**分析**：因为数据无序，最直接的方法是逐个比较。

**答案/结论**：顺序查找强调简单可靠。

### 例2：二分查找目标值
**层次**：关键例题

**题目**：给定升序数组和目标 x，输出其位置。

**分析**：比较 a[mid] 与 x 的大小关系，就能决定保留哪一半。

**答案/结论**：每轮都要让搜索区间明显缩小。

### 例3：找第一个大于等于 x 的位置
**层次**：易错例题

**题目**：给定有序数组，找出第一个满足 a[i] >= x 的下标。

**分析**：即使 mid 满足条件，也不能立刻返回，因为左边可能还有更早的位置。

**答案/结论**：边界题要学会保留“可能更优”的一侧。

## 练习

1. 顺序查找数组中的最小值位置。
2. 在有序数组中判断目标值是否存在。
3. 在有序数组中找最后一个小于等于 x 的位置。
4. 思考：什么时候“先排序再二分”比直接顺序查找更划算？

## 答案

1. 遍历数组并维护当前最小值位置。
2. 存在性判断是二分的基础应用。
3. 这是另一种常见边界二分。
4. 当要多次查询时，先排序的代价通常值得。
