# 信息学专题：并查集

> 信息学 | 数据结构 | 连通性维护

---

## 本课学什么

并查集用于维护“哪些点属于同一个集合”。它特别适合处理**动态连通性、分组归属、是否同类**这类问题。

本课重点掌握：

- 并查集维护的是什么信息；
- `find` 与 `merge` 分别在做什么；
- 路径压缩为什么能让查找更快。

---

## 核心知识 / 核心方法

### 1. 并查集维护的本质

每个集合选一个“代表元”。

- `find(x)`：找到 `x` 所在集合的代表；
- `merge(x, y)`：把两个集合合并。

若两个元素代表元相同，就说明它们在同一个集合。

### 2. 初始化

```cpp
vector<int> fa(n + 1);
for (int i = 1; i <= n; ++i) fa[i] = i;
```

一开始每个点各自单独成集合。

### 3. 查找 + 路径压缩

```cpp
int find(int x) {
    if (fa[x] == x) return x;
    return fa[x] = find(fa[x]);
}
```

路径压缩的作用：查找过后，把沿途节点直接挂到代表元下面，后续查询更快。

### 4. 合并

```cpp
void merge(int x, int y) {
    x = find(x);
    y = find(y);
    if (x != y) fa[x] = y;
}
```

若要进一步优化，还可按秩合并 / 按大小合并。

### 5. 典型应用

- 判断两点是否连通；
- 统计连通块个数；
- Kruskal 最小生成树；
- 朋友圈、家族关系、等价类问题。

---

## 易错点 / 关键提醒

1. **合并前忘记先找根**。
2. **把父节点是否相等误当成集合是否相同**，应比较根节点。
3. **不会数连通块**，其实统计 `fa[i] == i` 的根数即可。
4. **把并查集用于最短路**，这是模型误判。

---

## 例题

### 例1：基础例题——判断两点是否属于同一集合

若 `find(a) == find(b)`，则说明二者在同一集合。

### 例2：应用例题——统计朋友圈数量

给若干朋友关系，每次把两人所在集合合并。最后根节点个数，就是朋友圈数量。

---

## 练习题

1. 并查集最常维护哪类信息？
2. 路径压缩为什么能提速？
3. 为什么合并时必须先 `find`？

## 答案

1. 元素所属集合及集合间连通关系。
2. 因为它缩短了树高，让后续查找更接近常数。
3. 因为真正需要连接的是两个集合的代表元，而不是任意中间节点。
