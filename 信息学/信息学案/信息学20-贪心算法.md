# 信息学20：贪心算法

> 信息学｜算法设计方法

## 本课学什么

贪心不是“看起来顺手就选最大的或最小的”，而是每一步都依据某种原则做当前最优选择，并且要能说明这种选择为什么不会吃亏。

## 学习目标

- 理解贪心的局部最优思想。
- 能识别排序、局部决策、交换论证等常见贪心信号。
- 知道不是所有最优问题都能贪心。

## 概念

### 1. 贪心
贪心算法在每一步都根据当前信息做一个局部最优选择，希望最终得到全局最优。

### 2. 正确性证明
贪心难点不在“怎么选”，而在“为什么这样选一定正确”。交换论证和反证法是常见证明思路。

### 3. 适用信号
题目要求最多、最少、最优，且一旦选择后通常不需要回头修改时，常值得尝试贪心。

## 算法思想 / 分析思路

- 若题目常常要“先排序再做”，要警惕它可能是贪心题。
- 贪心标准要抓住真正影响后续空间的量，而不是表面上最大的量。
- 若当前决策会复杂影响未来，问题可能更适合 DP。

## 实现步骤

1. 分析题目目标，找出可能的局部最优标准。
2. 尝试用小例子验证这种标准是否合理。
3. 若可行，再思考为什么这种选择不会损失最优性。
4. 实现时常需先排序，再按规则线性扫描。

## 常见错误

1. **把局部最优误当全局最优**：很多题直觉上像贪心，但实际并不成立。
2. **只会排序，不会解释为什么这样排**：代码正确不代表思路真的可靠。
3. **忽略边界条件**：区间端点是否能重合等细节会影响判断。
4. **把适合 DP 的题硬做成贪心**：当前选择若会复杂影响未来，就要谨慎。

## 例题

### 例1：活动安排
**层次**：基础例题

**题目**：每个活动有开始和结束时间，求最多能参加多少个互不重叠活动。

**分析**：按结束时间从早到晚排序，优先选结束早的活动，为后面留下更多可选空间。

**答案/结论**：活动安排是最经典的贪心模型之一。

### 例2：为什么不是按开始时间最早选
**层次**：关键例题

**题目**：说明活动安排问题中，按开始时间最早排序为什么不一定最优。

**分析**：开始早但持续很久的活动，可能挡住后面很多短活动；结束早才真正保留了后续空间。

**答案/结论**：贪心标准必须抓住“影响后续空间”的关键量。

### 例3：找零钱的反例
**层次**：易错例题

**题目**：面值为 1、3、4，凑出 6。若每次取最大面值，为什么会失败？

**分析**：贪心会得到 4+1+1，但最优解是 3+3，说明“每次选最大”并不总对。

**答案/结论**：不是所有最优问题都能贪心。

## 练习

1. 贪心算法最核心的风险是什么？
2. 活动安排为什么通常按结束时间排序？
3. 什么情况下一个问题更可能适合 DP 而不是贪心？
4. 请举出一个“看起来能贪心，实际上不能”的信号。

## 答案

1. 局部最优选择未必能推出全局最优。
2. 因为结束越早，给后面活动留下的空间越大。
3. 当当前决策会复杂影响未来时，更可能适合 DP。
4. 例如眼前选最大收益会阻碍后续多个中等收益方案。
