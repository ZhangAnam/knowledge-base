# 考点5：指令流水线

> 信息学｜计算机体系结构基础｜重点理解“并行的是阶段，不是单条指令本身”

---

## 本课学什么

本课学习流水线为什么能提高处理器吞吐率，以及流水线中最常见的三类冲突：**结构相关、数据相关、控制相关**。  
难点是分清：**流水线提高的是单位时间内完成的指令数量，不一定缩短单条指令的总延迟**。

学完后，你应该能够：

- 说出经典五级流水线的基本阶段；
- 判断三类流水线冒险（hazard）的来源；
- 理解暂停、前递、分支预测等常见处理方法。

---

## 核心知识

### 1. 为什么要用流水线

如果一条指令必须完整执行完，下一条指令才能开始，那么很多硬件部件会在大量时间里闲置。  
流水线的思路是：**把指令执行拆成多个阶段，让不同指令同时处在不同阶段中工作**。

这有点像工厂流水作业：

- 第一件产品在第二道工序时，
- 第二件产品已经可以进入第一道工序。

所以流水线提高的是**吞吐率**。

### 2. 经典五级流水线

常见阶段可写成：

```text
取指 IF → 译码 ID → 执行 EX → 访存 MEM → 写回 WB
```

各阶段大意如下：

| 阶段 | 含义 |
| --- | --- |
| IF | 从主存取出指令 |
| ID | 分析指令、读寄存器 |
| EX | 做运算或计算有效地址 |
| MEM | 访问数据存储器 |
| WB | 把结果写回寄存器 |

### 3. 吞吐率与加速比

理想情况下：

- 非流水方式：每条指令都要完整走完全部阶段；
- 流水方式：装满流水线后，每个时钟周期可能完成一条指令。

因此理想状态下，若有 `k` 个阶段，长指令流的加速比可以接近 `k`。  
但现实中总会受到阶段不均衡、寄存器开销、冲突停顿等影响。

### 4. 流水线中的三类冒险

#### 结构相关（Structure Hazard）

多个阶段同时争用同一硬件资源。

例如：

- 取指和访存都要访问同一存储器；
- 同时需要同一个运算部件。

常见解决方法：

- 增加硬件资源；
- 指令和数据分离存储；
- 对冲突操作进行暂停。

#### 数据相关（Data Hazard）

后一条指令依赖前一条指令的结果，但结果还没准备好。

例如：

```text
ADD R1, R2, R3
SUB R4, R1, R5
```

第二条指令要用 `R1`，但第一条指令可能还没写回。

常见解决方法：

- **前递 / 转发（forwarding）**：直接把前一阶段结果送给后一条指令；
- **暂停（stall）**：插入气泡等待数据就绪；
- 编译器调整指令顺序。

#### 控制相关（Control Hazard）

分支、跳转会导致“下一条该取哪条指令”暂时不确定。

例如条件跳转：

```text
if (x > 0) goto L1;
```

在条件尚未判断出来前，后续取指可能取错。

常见解决方法：

- 暂停等待分支结果；
- **分支预测**；
- 延迟槽（较老架构常见）。

### 5. CPI 与流水线停顿

理想流水线常说 **CPI ≈ 1**，即平均每条指令约用一个时钟周期完成输出。  
但一旦发生冒险并插入停顿，实际 CPI 就会变大。

理解重点：

- **停顿越多，流水线效率越低**；
- 流水线性能不只看级数，还看冒险控制得好不好。

### 6. 超标量与动态调度

在流水线基础上，处理器还可能进一步提高并行性：

- **超标量**：一个周期发射多条指令；
- **动态调度**：运行时根据资源和依赖关系安排执行顺序；
- **乱序执行**：不必严格按程序顺序完成内部执行，但对外结果仍需正确。

这部分在入门层面重点记住：  
**它们都是为了进一步提高指令级并行性。**

---

## 易错点 / 关键提醒

1. **流水线提高的是吞吐率，不一定降低单条指令延迟**。

2. **数据相关不是“数据错了”，而是“需要的数据还没到”**。

3. **控制相关的根源是分支导致取指方向不确定**，不是单纯“跳转指令比较慢”。

4. **CPI=1只是理想情况**  
   实际流水线会因停顿、预测失败等原因变大。

5. **前递不能解决所有数据相关**  
   某些场景仍然需要停顿。

## 例题

### 例1：判断冒险类型

若一台机器使用统一存储器，取指阶段和访存阶段不能同时访问内存，这属于哪一类流水线冒险？

#### 解析

这是两个阶段争用同一硬件资源，属于结构相关。

#### 答案

**结构相关**

### 例2：分析数据相关

已知指令序列：

```text
ADD R1, R2, R3
SUB R4, R1, R5
```

第二条指令为什么可能需要等待？

#### 解析

因为 `SUB` 要读取 `R1`，而 `R1` 是前一条 `ADD` 的结果，可能还未写回寄存器。

#### 答案

**因为后一条指令依赖前一条指令尚未准备好的结果，属于数据相关。**

### 例3：理解分支预测

为什么分支预测可以提高流水线效率？

#### 解析

分支未决时，如果完全停住流水线，吞吐率会下降。  
预测机制让处理器先按一个方向继续取指，正确时就能减少停顿。

#### 答案

**因为它减少了控制相关带来的等待时间。**

### 例4：判断说法正误

“流水线级数越多，处理器一定越快。”这句话对吗？

#### 解析

不对。级数过多会带来更复杂的控制、更大的寄存器开销和更严重的分支代价。

#### 答案

**错误。**

## 练习题

1. 经典五级流水线通常包括哪五个阶段？  
2. 结构相关、数据相关、控制相关分别是因为什么产生的？  
3. 为什么分支指令容易引起控制相关？  
4. 前递主要用于缓解哪一类问题？  
5. 理想流水线中常说 CPI 约为多少？

---

## 答案

1. **取指、译码、执行、访存、写回**。  
2. **结构相关源于硬件资源冲突；数据相关源于指令间数据依赖；控制相关源于分支跳转导致取指方向不确定**。  
3. **因为在分支结果未确定前，后续该取哪条指令不明确**。  
4. **数据相关**。  
5. **约为 1**。
