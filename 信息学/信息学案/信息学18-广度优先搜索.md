# 信息学18：广度优先搜索

> 信息学｜图搜索与最短步数基础

## 本课学什么

BFS 的核心是“从起点一层一层向外扩展”。只要每条边代价相同，第一次到达某个状态时，所用步数就是最少的。

## 学习目标

- 理解 BFS 的层次扩展过程。
- 掌握队列、访问标记和距离数组的配合。
- 能处理无权图最短路、迷宫最少步数和多源扩散问题。

## 概念

### 1. BFS
广度优先搜索按层扩展：距离起点 1 步的状态先被访问，2 步的状态后访问。

### 2. 队列与距离
队列负责保持“先发现的先处理”，距离数组负责记录每个状态到起点的最短步数。

### 3. 多源 BFS
若有多个起点，可以把它们同时看作第 0 层，一起入队开始扩展。

## 算法思想 / 分析思路

- 当题目问“最少几步”“最短边数”且边权相同，优先想到 BFS。
- BFS 的正确性来自按层扩展，不是来自某个公式。
- 多源 BFS 只是初始化不同，本质仍是按层推进。

## 实现步骤

1. 明确状态表示，如点编号或坐标。
2. 初始化队列、vis 和 dist。
3. 把起点（或多个起点）入队并立即标记。
4. 循环取出队头，扩展所有相邻状态。
5. 第一次到达终点时即可得到最短步数。

## 常见错误

1. **重复入队**：vis 应在入队时设置，而不是出队时。
2. **把 BFS 用在带权最短路**：普通 BFS 只适用于边权相同的情况。
3. **距离没有初始化**：未访问状态应有明确初值，如 -1 或 INF。
4. **混淆图状态与网格状态**：点编号和坐标的表示方式不同。

## 例题

### 例1：无权图单源最短路
**层次**：基础例题

**题目**：求点 1 到其余各点的最短边数。

**分析**：BFS 的层数就是最短边数，因为它总是先处理离起点更近的点。

**答案/结论**：dist[v] = dist[u] + 1 体现的是“再走一步”。

### 例2：迷宫最少步数
**层次**：关键例题

**题目**：在网格中从起点走到终点，每次只能上下左右移动一步，求最少步数。

**分析**：迷宫本质上也是图，每个格子是状态，相邻可走格子之间连边。

**答案/结论**：网格最短步数题通常是 BFS 模型。

### 例3：多源扩散
**层次**：易错例题

**题目**：有多个起火点，火同时向四周蔓延，求每个格子第一次着火时间。

**分析**：所有火源的初始时间都为 0，并且一开始全部入队。

**答案/结论**：多源 BFS 的关键是“多个起点同时作为第 0 层”。

## 练习

1. 为什么说 BFS 第一次到达某点时路径最短？
2. 为什么 vis 一般在入队时就设为 true？
3. 迷宫最少步数题为什么更适合 BFS 而不是 DFS？
4. 多源 BFS 和单源 BFS 的区别主要体现在哪一步？

## 答案

1. 因为 BFS 按距离从小到大一层一层扩展。
2. 为了防止同一状态重复入队。
3. 因为 BFS 天然按步数推进，第一次到终点就是最短。
4. 主要体现在初始化：多个起点一起入队，距离设为 0。
