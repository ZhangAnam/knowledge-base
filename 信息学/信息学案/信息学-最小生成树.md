# 信息学专题：最小生成树

> 信息学 | 图论 | 连通代价最小化

---

## 本课学什么

最小生成树研究的是：在保证所有点连通的前提下，怎样选出总代价最小的一组边。

本课重点掌握：

- 生成树与最小生成树的含义；
- Kruskal 与 Prim 的核心思想；
- 为什么“选最小边”不是无脑乱选，而要满足不成环。

---

## 核心知识 / 核心方法

### 1. 生成树的含义

对一个连通图，若选出若干边，使得：

- 所有点都连通；
- 没有环；
- 边数恰好是 `n - 1`；

这组边就构成一棵生成树。

最小生成树是在所有生成树中，总边权最小的那一棵。

### 2. Kruskal

核心想法：按边权从小到大排序，能加就加，但不能形成环。

```cpp
sort(edges.begin(), edges.end());
for (auto [w, u, v] : edges) {
    if (find(u) != find(v)) {
        merge(u, v);
        ans += w;
        ++cnt;
    }
}
```

**为什么要配合并查集**：要快速判断加这条边会不会让两个已经连通的点再次连起来，从而形成环。

### 3. Prim

核心想法：从一个点出发，每次选一条把“树内点”和“树外点”连接起来的最小边。

它更像“逐步扩张一个连通块”。

### 4. 两种算法的使用感觉

- Kruskal：像“从全局边集中挑边”；
- Prim：像“从当前树向外扩展”。

---

## 易错点 / 关键提醒

1. **只顾选最小边，不检查是否成环**。
2. **图本身不连通**，这时不存在生成全图的 MST。
3. **Kruskal 中并查集判断写错**。
4. **把最短路和最小生成树混淆**，二者目标完全不同。

---

## 例题

### 例1：基础例题——修路问题

给若干城市和修路代价，要求所有城市最终连通且总花费最小，这就是 MST 标准模型。

### 例2：关键例题——为什么最短路树不一定是最小生成树

最短路强调“从某个起点到各点距离最小”，MST 强调“全图连通总边权最小”，目标不同。

---

## 练习题

1. Kruskal 每次选边前必须检查什么？
2. Prim 和 Kruskal 的主要思路差别是什么？
3. 为什么图不连通时无法得到最小生成树？

## 答案

1. 检查是否会形成环。
2. Prim 是从点集向外扩张，Kruskal 是从全局最小边开始筛选。
3. 因为无法用 `n - 1` 条边把所有点连成一棵树。
