# 信息学6：函数

> 信息学｜基础编程入门

## 本课学什么

函数的作用不是“让代码看起来高级”，而是把重复使用的逻辑封装起来，让程序更清楚、更容易检查。

## 学习目标

- 理解函数由函数名、参数、返回值三部分构成。
- 会把重复操作提炼成函数。
- 能区分“函数做什么”和“主函数怎样调用”。

## 概念

### 1. 函数
函数可以理解为“完成某项固定任务的小工具”。调用函数，就是把问题交给这个工具处理。

### 2. 参数与返回值
参数是函数输入的信息，返回值是函数输出的结果。不是所有函数都需要返回值，但每个函数都要职责明确。

### 3. 作用域
函数内部定义的局部变量只在本函数中有效。不同函数之间不要依赖碰巧同名的变量。

## 算法思想 / 分析思路

- 发现重复代码时，先想能否提炼成函数。
- 写函数前先说清楚：输入是什么，输出是什么。
- 函数越短、职责越清晰，越不容易出错。

## 实现步骤

1. 确定函数要完成的任务。
2. 设计参数列表和返回值类型。
3. 在函数内部实现具体逻辑。
4. 在 main 中调用函数并接收结果。
5. 用简单样例单独测试函数。

## 常见错误

1. **忘记返回值**：声明了非 void 函数，却没有 return。
2. **参数类型不匹配**：调用时传入的数据类型与函数定义不一致。
3. **把局部变量当外部变量使用**：函数执行结束后，局部变量的作用域就结束了。
4. **函数职责过多**：一个函数里什么都做，调试会很困难。

## 例题

### 例1：写一个求两数较大值的函数
**层次**：基础例题

**题目**：定义函数 max2(a,b)，返回两个整数中的较大值。

**分析**：这个函数只做一件事：比较并返回结果，是函数入门题的标准模型。

**答案/结论**：用 return 输出函数结果。

### 例2：封装“判断闰年”逻辑
**层次**：关键例题

**题目**：把闰年判断写成 bool 函数 isLeap(year)。

**分析**：把复杂条件单独封装后，主程序读起来更清楚，也便于复用。

**答案/结论**：返回 true 或 false 比直接输出文字更灵活。

### 例3：计算各位数字之和
**层次**：易错例题

**题目**：定义函数 digitSum(n)，返回正整数各位数字之和。

**分析**：函数内部可用 while + 取模，但要注意 n 是参数副本。

**答案/结论**：函数能把“拆位求和”这类常见步骤独立出来。

## 练习

1. 定义函数 square(x)，返回 x 的平方。
2. 定义函数 even(n)，判断 n 是否为偶数。
3. 定义函数 area(r)，返回半径为 r 的圆面积。
4. 尝试把“交换两个数”写成一个函数。

## 答案

1. 平方是 x * x。
2. 偶数判断仍可用 n % 2 == 0。
3. 若涉及小数，返回值宜用 double。
4. 若想真正交换外部变量，需要进一步学习引用传参。
