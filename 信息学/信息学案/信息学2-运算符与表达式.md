# 信息学2：运算符与表达式

> 信息学｜基础编程入门

## 本课学什么

本课学习程序如何“把数据算起来”。你不仅要会写加减乘除，更要理解优先级、整型除法、取模和逻辑表达式的真正含义。

## 学习目标

- 认识算术、关系、逻辑三类常用运算符。
- 能正确分析表达式的运算顺序。
- 会用括号改写复杂表达式，减少歧义。

## 概念

### 1. 算术运算符
+、-、*、/、% 是最常见的算术运算符，其中 % 只适用于整数，常用于奇偶、整除和拆位问题。

### 2. 关系与逻辑运算
关系运算比较大小，结果只有真或假；逻辑运算用于把多个条件组合起来。

### 3. 表达式
表达式是“值的计算规则”。同样的变量，如果优先级理解错，结果会完全不同。

## 算法思想 / 分析思路

- 先把题目中的数学关系写清楚，再翻译成程序表达式。
- 复杂表达式宁可多写括号，也不要依赖模糊记忆。
- 当题目问“是否满足条件”时，要把自然语言转成关系表达式和逻辑表达式。

## 实现步骤

1. 把题目中的公式写成数学式。
2. 判断除法是否需要保留小数。
3. 按优先级写成程序表达式，必要时补括号。
4. 用简单样例手算一次，核对程序表达式。

## 常见错误

1. **把 / 当成实数除法**：两个 int 相除时，小数部分会被截掉。
2. **误用 % 处理小数**：取模只适用于整数。
3. **逻辑条件少括号**：and 与 or 混用时容易改变原意。
4. **忽略优先级**：a + b * c 和 (a + b) * c 的结果不同。

## 例题

### 例1：求三数平均值
**层次**：基础例题

**题目**：输入 a、b、c，输出平均值。

**分析**：平均值本质是总和除以 3，若要保留小数，除法必须进入浮点运算。

**答案/结论**：可先求和，再用 sum / 3.0。

### 例2：判断一个数是否为偶数
**层次**：关键例题

**题目**：输入整数 n，判断它是否为偶数。

**分析**：偶数的本质是能被 2 整除，因此可以用取模表达。

**答案/结论**：条件可写成 n % 2 == 0。

### 例3：年龄在 10 到 15 岁之间
**层次**：易错例题

**题目**：输入 age，判断它是否满足 10≤age≤15。

**分析**：程序中不能直接连写比较，要拆成两个条件再合并。

```text
age >= 10 && age <= 15
```

**答案/结论**：自然语言的连写比较应改成两个关系式。

## 练习

1. 输入长和宽，输出矩形周长。
2. 输入整数 n，输出 n 的个位数字。
3. 输入 x，判断它是否同时大于 0 且小于 100。
4. 输入总秒数，换算成“分和秒”。

## 答案

1. 周长公式要写成 2 * (a + b)。
2. 个位数字可用 n % 10。
3. 条件可写成 x > 0 && x < 100。
4. 分是 s / 60，秒是 s % 60。
