# 信息学8：排序

> 信息学｜基础算法

## 本课学什么

排序不是为了“把数排整齐”而已，更重要的是为后续查找、去重、贪心和区间处理创造条件。本课重点是理解排序的用途、常见方法和使用时机。

## 学习目标

- 理解升序、降序和按关键字排序的含义。
- 会描述冒泡、选择和库函数 sort 的基本思路。
- 能判断什么时候应先排序，再做后续处理。

## 概念

### 1. 排序
排序是按照某种规则重新安排元素顺序，常见规则有从小到大、从大到小、按字典序或按关键字。

### 2. 简单排序与库函数排序
手写简单排序有助于理解比较与交换；在实际竞赛中，更常用高效的库函数 sort。

### 3. 排序的价值
很多题目本身不是排序题，但一旦排好序，去重、二分、贪心、区间统计都会更方便。

## 算法思想 / 分析思路

- 先想清排序规则：是按值、按长度、按分数还是按字典序。
- 如果排序后要继续处理，必须重新理解“位置”与“元素”的关系。
- 题目出现“从小到大考虑”“相同元素放在一起”时，排序常常是突破口。

## 实现步骤

1. 明确题目要升序还是降序。
2. 若使用 sort，确认排序区间和比较器。
3. 排序完成后，根据新顺序继续完成去重、选前几项或统计。
4. 用包含重复值和边界值的数据测试结果。

## 常见错误

1. **循环边界写错**：手写排序时最容易出现越界。
2. **排序方向反了**：题目要降序却写成默认升序。
3. **排序后仍按原下标理解题意**：一旦重新排序，原位置意义往往已经失效。
4. **误把排序当终点**：很多题真正目标是排序后的进一步处理。

## 例题

### 例1：排序后去重输出
**层次**：基础例题

**题目**：输入 n 个整数，按从小到大排序，并去掉重复元素后输出。

**分析**：先排序可以让相同元素相邻，再顺次扫描，只输出和前一个不同的值。

**答案/结论**：排序常常是为去重服务。

### 例2：输出前三大数
**层次**：关键例题

**题目**：输入一组整数，按从大到小输出前三个。

**分析**：这是“排序后截取前缀”的典型应用。

**答案/结论**：降序排序后取前 3 个元素即可。

### 例3：最接近的两个数
**层次**：易错例题

**题目**：求数组中差值最小的两个数。

**分析**：排序后只需比较相邻元素；若不排序，要考虑任意两两组合。

**答案/结论**：很多“最接近”问题都要先想到排序。

## 练习

1. 用 sort 将一组整数按升序输出。
2. 排序后统计不同整数的个数。
3. 将若干字符串按字典序排序。
4. 排序后求中位数，并思考 n 为偶数和奇数时的区别。

## 答案

1. 默认 sort 为升序。
2. 排序后当 a[i] != a[i-1] 时计数。
3. 字符串也支持按字典序排序。
4. 中位数位置依赖排序后的下标。
