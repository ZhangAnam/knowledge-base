# 信息学19：递归与分治

> 信息学｜算法设计基础

## 本课学什么

递归不是“函数自己调自己”这么简单，而是把大问题交给规模更小的同类问题。分治则进一步强调“分开解决，再把结果合起来”。

## 学习目标

- 掌握递归的函数含义、边界和递归关系。
- 理解分治的“分—治—合”结构。
- 会判断什么时候递归高效，什么时候会出现重复计算。

## 概念

### 1. 递归
如果一个问题可以拆成更小规模的同类问题，就可能适合递归。递归必须有清晰的函数含义和边界。

### 2. 返回过程
递归不仅有“往下调用”，还有“往上返回”。很多答案正是在返回时逐层合并出来的。

### 3. 分治
分治强调把原问题拆成若干子问题分别求解，最后再合并结果。归并排序就是典型例子。

## 算法思想 / 分析思路

- 写递归前先说清楚函数到底表示什么。
- 若子问题之间互相独立、结果还需要合并，常是分治信号。
- 当递归存在大量重复子问题时，要考虑记忆化或 DP。

## 实现步骤

1. 定义递归函数的含义。
2. 写出最小规模时的边界答案。
3. 说明当前问题怎样转化为更小问题。
4. 若是分治，再设计子问题结果的合并方式。
5. 检查每次递归是否真的让规模变小。

## 常见错误

1. **边界漏写**：这是死递归和栈溢出的根源。
2. **规模没有缩小**：区间或参数不变，就会反复调用同一状态。
3. **只会往下调，不会解释返回过程**：对子问题如何合成整体答案没有理解。
4. **把所有问题都递归化**：有些递归存在大量重复计算，效率很低。

## 例题

### 例1：递归求阶乘
**层次**：基础例题

**题目**：定义 fac(n)，求 n!。

**分析**：这里最重要的是三件事：fac(n) 的含义、fac(0)=1 的边界、fac(n)=n*fac(n-1) 的递归关系。

**答案/结论**：阶乘是最典型的递归入门模型。

### 例2：归并排序为什么属于分治
**层次**：关键例题

**题目**：说明归并排序中的“分、治、合”分别是什么。

**分析**：不断二分是“分”，递归排好左右两段是“治”，merge 把两段有序区间合成整体有序是“合”。

**答案/结论**：分治的关键常常不只在拆，还在怎么合并。

### 例3：二分查找递归死循环
**层次**：易错例题

**题目**：为什么把区间递归写成 dfs(l, mid) 和 dfs(mid, r) 有时会死循环？

**分析**：当区间长度很小时，mid 可能等于 l 或 r，导致规模没有继续缩小。

**答案/结论**：递归区间更新必须保证严格缩小。

## 练习

1. 设计递归前最先应明确哪三件事？
2. 为什么说归并排序是分治而不仅仅是递归？
3. 递归返回过程在解决问题时起什么作用？
4. 什么情况下应考虑把递归改成记忆化或迭代？

## 答案

1. 函数含义、边界条件、递归关系。
2. 因为它不仅递归，还要把子问题答案进行合并。
3. 很多整体答案是在返回时逐层合成的。
4. 当有大量重复子问题或递归层数过深时。
