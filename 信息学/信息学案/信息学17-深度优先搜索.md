# 信息学17：深度优先搜索

> 信息学｜图搜索与回溯基础

## 本课学什么

DFS 的核心不是“会写递归模板”，而是理解它为什么会沿着一条路尽量走深，走不通再返回。很多连通块、路径搜索、全排列问题都建立在这种搜索节奏上。

## 学习目标

- 理解 DFS 的搜索顺序与回溯思想。
- 会在图、网格和排列枚举中识别 DFS 模型。
- 知道什么时候需要 vis 标记，什么时候需要撤销选择。

## 概念

### 1. DFS
深度优先搜索会先沿一条路径不断走深，直到不能继续，再退回上一个状态尝试其他分支。

### 2. 回溯
回溯是“做出选择—递归深入—再撤销选择”的过程，是 DFS 在方案枚举题中的核心。

### 3. 访问标记
在图和网格中，vis 用来避免重复访问同一状态，尤其是有环时非常重要。

## 算法思想 / 分析思路

- 若题目强调“先走一条路到底”，可以考虑 DFS。
- 图上的 DFS 重点是防止重复访问；回溯型 DFS 重点是做完要撤销。
- DFS 适合找可达性、连通块和枚举所有方案，但不保证第一次找到的是最短路。

## 实现步骤

1. 明确当前状态是什么，如点编号或网格坐标。
2. 写出递归边界或终止条件。
3. 进入状态后先标记或记录选择。
4. 枚举下一步可走的状态并递归。
5. 若属于回溯题，在返回前撤销选择。

## 常见错误

1. **先递归后标记**：图中可能形成环，导致死递归。
2. **回溯时忘记撤销**：会让后续方案缺失或状态污染。
3. **把 DFS 当最短路算法**：DFS 可以找到路，但不保证第一次找到的最短。
4. **边界判断放得太晚**：网格题应先判越界、障碍、是否访问过。

## 例题

### 例1：统计连通块个数
**层次**：基础例题

**题目**：给定无向图，求有多少个连通块。

**分析**：从每个未访问点出发做一次 DFS，一次 DFS 会扫完整个连通块。

**答案/结论**：启动 DFS 的次数就是连通块个数。

### 例2：输出从起点到终点的一条路径
**层次**：关键例题

**题目**：在图中寻找从 1 到 n 的一条可行路径。

**分析**：递归栈天然记录了当前路径，进入节点时加入路径，失败返回时弹出。

**答案/结论**：这里要体会“递归栈就是当前搜索路径”。

### 例3：全排列中的撤销选择
**层次**：易错例题

**题目**：为什么全排列 DFS 中必须在返回时恢复 used 数组？

**分析**：若不撤销，本轮选择会错误地影响后续分支，导致方案缺失。

**答案/结论**：回溯题中“恢复现场”和图搜索中的 vis 作用不同。

## 练习

1. 为什么图上 DFS 通常要配合 vis 数组？
2. 网格 DFS 中，哪些判断应放在递归前完成？
3. 为什么 DFS 不适合直接求无权图最短路？
4. 全排列 DFS 中，“做选择”和“撤销选择”分别对应什么？

## 答案

1. 因为图中可能有环，不标记会重复访问。
2. 越界、障碍、是否访问过等条件应先判断。
3. 因为 DFS 先找到的路径不一定步数最少。
4. 例如 used[i]=true 是做选择，used[i]=false 是撤销选择。
