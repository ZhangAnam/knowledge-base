# 考点4：CPU结构与数据通路

## 本课学什么

本课要弄清楚三件事：**CPU由哪些核心部件组成、数据在CPU内部怎样流动、执行一条指令时各部件如何协同工作**。  
难点不在于背部件名称，而在于分清“谁负责运算、谁负责控制、谁负责暂存、谁负责传送”。

学完后，你应该能够：

- 说清运算器、控制器、寄存器组的作用；
- 判断单总线、双总线、多总线结构的特点；
- 用“取指—分析—执行”的视角描述一条指令的执行过程。

## 核心知识

### 1. CPU的基本组成

CPU通常可以看成由**运算器、控制器、寄存器组**三部分构成。

#### 运算器

运算器负责真正“算”和“改”数据，常见部件有：

- **ALU（算术逻辑单元）**：完成加、减、与、或、比较、移位等操作；
- **ACC（累加器）**：保存运算过程中的中间结果；
- **PSW / 标志寄存器**：保存零标志、进位标志、溢出标志等状态信息。

#### 控制器

控制器负责“指挥”。

- **PC（程序计数器）**：存放下一条指令的地址；
- **IR（指令寄存器）**：存放当前正在执行的指令；
- **指令译码器**：把操作码翻译成具体控制信号；
- **时序控制逻辑**：按照时钟节拍安排各部件何时工作。

#### 寄存器组

寄存器是CPU内部最快的存储单元，用来临时保存数据、地址或控制信息。

- **通用寄存器**：可存放操作数或中间结果；
- **专用寄存器**：如PC、IR、MAR、MDR等，职责更明确。

### 2. 常见辅助寄存器

在分析数据通路时，经常会遇到：

| 寄存器 | 作用 |
| --- | --- |
| PC | 存放下一条指令地址 |
| IR | 存放当前指令 |
| MAR | 存放要访问的主存地址 |
| MDR | 存放从主存读出或将写入主存的数据 |

其中最容易混淆的是：

- **MAR装地址，不装数据**；
- **MDR装数据，不装地址**。

### 3. 什么是数据通路

数据通路就是**数据在CPU内部和CPU与主存之间移动的路线**。  
它决定了“一个时钟节拍内，哪些数据能并行流动，哪些操作必须分步完成”。

常见的数据通路设计方式如下。

#### 单总线结构

- 所有部件共用一条总线；
- 结构简单，成本低；
- 同一时刻只能有较少数据传送，容易形成冲突。

#### 双总线结构

- 比单总线并行能力更强；
- 常用于兼顾成本和性能的设计；
- 比单总线更容易安排“取数+送数”等操作。

#### 三总线或多总线结构

- 可支持更多部件并行传送数据；
- 性能更高；
- 控制更复杂、硬件成本更高。

### 4. 指令执行过程与数据流动

一条指令通常经历以下阶段：

1. **取指**：根据PC到主存取出指令；
2. **分析/译码**：把指令送入IR并识别操作类型；
3. **执行**：完成运算、访存或转移；
4. **写回**：把结果写回寄存器或主存。

一个典型的取指过程可以概括为：

```text
PC → MAR
主存[MAR] → MDR
MDR → IR
PC ← PC + 指令长度
```

这说明：

- **PC提供地址**；
- **MAR负责送地址到主存**；
- **MDR负责接收主存返回的数据**；
- **IR保存当前指令，供译码器分析**。

### 5. 时钟周期、机器周期、指令周期

这是体系结构中常见的层次关系。

| 名称 | 含义 |
| --- | --- |
| 时钟周期 | 处理器最基本的时间单位 |
| 机器周期 | 完成一个基本操作所需的时间，通常由若干时钟周期组成 |
| 指令周期 | 执行一条指令所需的全部时间 |

要点：

- 一条复杂指令通常不止一个时钟周期；
- **主频越高，时钟周期越短**；
- 但主频高不等于一定更快，还要看每条指令平均需要多少周期。

---

## 易错点 / 关键提醒

1. **不要把控制器理解成“只负责译码”**  
   它还负责发出控制信号，组织整个执行过程。

2. **不要把寄存器和主存混为一谈**  
   寄存器在CPU内部，速度极快；主存在CPU外部，容量更大但更慢。

3. **PC中存的是“下一条指令地址”**  
   不是“当前已经执行完的指令地址”。

4. **总线越多不等于一定最好**  
   多总线提升并行性，但硬件复杂度和成本也会提高。

5. **指令周期和时钟周期不是同一个概念**  
   指令周期通常由多个时钟周期组成。

## 例题

### 例1：判断部件职责

下列说法中，哪一个正确？

A. ALU负责保存下一条指令地址  
B. PC负责保存下一条指令地址  
C. MDR负责保存指令译码结果  
D. IR负责保存主存地址

#### 解析

- ALU负责运算，不负责顺序控制；
- **PC保存下一条指令地址**；
- MDR保存数据，不保存译码结果；
- IR保存当前指令，不保存主存地址。

#### 答案

**B**

### 例2：分析取指阶段的数据流

某处理器执行取指操作时，需要把当前指令从主存送入IR。请写出主要的数据流顺序。

#### 解析

先用PC给出地址，再由主存读出指令，最后送入IR。

#### 答案

```text
PC → MAR
主存 → MDR
MDR → IR
PC ← PC + 1（或 + 指令长度）
```

### 例3：比较总线结构

为什么单总线结构虽然简单，却容易成为性能瓶颈？

#### 解析

因为多个部件共享同一条数据通路，同一时刻能完成的数据传送有限。  
如果既想取数据又想送结果，就容易发生争用，只能分时完成。

#### 答案

**原因是总线共享导致并行能力弱，部件之间容易发生传送冲突。**

### 例4：区分时间单位

“执行一条指令一定只需要一个时钟周期。”这句话对吗？

#### 解析

不对。执行一条指令通常要经历取指、译码、执行等多个步骤，往往对应多个时钟周期。

#### 答案

**错误。**

## 练习题

1. CPU中负责完成算术逻辑运算的核心部件是什么？  
2. MAR和MDR分别保存什么信息？  
3. 单总线结构的主要优点和主要缺点各是什么？  
4. 简述一条指令执行时“取指—译码—执行”的基本含义。  
5. 指令周期、机器周期、时钟周期三者中，哪一个时间粒度最小？

---

## 答案

1. **ALU（算术逻辑单元）**。  
2. **MAR保存主存地址，MDR保存主存读写的数据**。  
3. **优点：结构简单、成本低；缺点：并行性差，容易发生总线冲突**。  
4. **取指出当前指令，译码分析要做什么，执行阶段真正完成运算、访存或转移**。  
5. **时钟周期最小**。
