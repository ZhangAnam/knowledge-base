# 信息学专题：字符串 Hash

> 信息学 | 字符串算法 | 快速比较

---

## 本课学什么

字符串 Hash 的核心作用，是把“比较字符串”转化为“比较整数”，从而让子串比较、判重、回文判断等问题更高效。

本课重点掌握：

- 前缀 Hash 怎样构造；
- 为什么能在 `O(1)` 时间取子串 Hash；
- Hash 为什么快，以及为什么仍可能冲突。

---

## 核心知识 / 核心方法

### 1. 基本想法

给字符串的每个前缀计算一个“编码值”，让不同字符串大概率对应不同数字。

常见写法：

```cpp
using ull = unsigned long long;
const ull P = 131;
```

### 2. 前缀 Hash

```cpp
vector<ull> h(n + 1), p(n + 1);
p[0] = 1;
for (int i = 1; i <= n; ++i) {
    h[i] = h[i - 1] * P + s[i - 1];
    p[i] = p[i - 1] * P;
}
```

其中：

- `h[i]` 表示前 `i` 个字符的 Hash；
- `p[i]` 表示 `P` 的 `i` 次幂。

### 3. `O(1)` 取子串 Hash

```cpp
ull getHash(int l, int r) { // 取 s[l..r]
    return h[r + 1] - h[l] * p[r - l + 1];
}
```

这样就能快速比较两个子串是否相同。

### 4. Hash 的用途

- 子串相等判断；
- 字符串判重；
- 回文串判断（配合反串 Hash）；
- 二分答案 + 子串比较。

### 5. Hash 不是完全无误

Hash 是“高概率正确”的工具，不是数学上绝对无冲突。

常见减冲突方法：

- 用两个模数做双 Hash；
- 使用 `unsigned long long` 自然溢出；
- 关键场景下再做一次原串比较。

---

## 易错点 / 关键提醒

1. **下标混乱**，尤其是 `0` 下标与 `1` 下标混写。
2. **忘记预处理幂数组 `p[]`**。
3. **把 Hash 相等当作绝对相等**。
4. **反串下标换算错误**，做回文题时尤常见。

---

## 例题

### 例1：基础例题——比较两个子串是否相同

只要取出两段区间的 Hash，比较是否相等即可。

### 例2：应用例题——判断回文串

同时预处理原串和反串的前缀 Hash，把一段正向子串和对应反向子串比较即可。

## 练习题

1. 为什么前缀 Hash 可以在 `O(1)` 取子串 Hash？
2. 为什么 Hash 快速，但不保证绝对无冲突？
3. 做回文判断时，为什么常要同时维护正串和反串的 Hash？

## 答案

1. 因为前缀值和幂数组预处理后，子串值可由两段前缀直接相减得到。
2. 因为不同字符串可能偶然映射到同一个数。
3. 因为回文本质是正着读和倒着读相同。
