# 信息学16：图的存储

> 信息学｜图论基础

## 本课学什么

图论题往往不是先难在算法，而是先难在“图怎样存”。本课重点是理解邻接矩阵、邻接表、链式前向星分别适合什么场景，以及它们对遍历效率有什么影响。

## 学习目标

- 理解图的点、边以及有向图、无向图的基本表示。
- 掌握邻接矩阵、邻接表、链式前向星三种常见存储方式。
- 能根据点数、边数和操作需求选择合适表示。

## 概念

### 1. 图的存储任务
图的存储本质上是在回答两个问题：哪些点存在，哪些点之间有边，以及边上是否还带权值。

### 2. 邻接矩阵
用二维数组记录任意两点之间是否有边或边权。查询一条边是否存在很快，但空间开销通常较大。

### 3. 邻接表与链式前向星
当图较稀疏时，更常用“只存存在的边”。邻接表写法直观，链式前向星更常见于竞赛模板和大数据图题。

## 算法思想 / 分析思路

- 先看图是稠密还是稀疏，再选存储方式。
- 若经常询问“u 到 v 是否直接有边”，邻接矩阵更方便。
- 若主要任务是遍历每个点的出边，邻接表或链式前向星通常更合适。

## 实现步骤

1. 读题，确定点数、边数以及是否带权。
2. 根据数据规模选择存储结构。
3. 建图时注意有向图和无向图的加边方式不同。
4. 建图完成后，通过遍历相邻点验证图是否存对。

## 常见错误

1. **无向图只加一条边**：无向边通常要加两次，分别表示两个方向。
2. **邻接矩阵和邻接表场景混用**：边很少时还开 n×n 矩阵，空间可能浪费严重。
3. **点编号范围看错**：1 到 n 与 0 到 n-1 的写法不能混。
4. **带权图忘记保存边权**：只记录终点，不记录权值，后续最短路就无法使用。

## 例题

### 例1：用邻接矩阵表示班级关系图
**层次**：基础例题

**题目**：已知若干同学之间是否认识，如何快速判断两人是否直接认识？

**分析**：若点数不大且频繁查询某两点之间是否有边，邻接矩阵查询最直接。

**答案/结论**：邻接矩阵适合快速判断边是否存在。

### 例2：遍历稀疏图的所有出边
**层次**：关键例题

**题目**：图中点很多、边较少，要求遍历每个点的相邻点。

**分析**：此时若用邻接矩阵会浪费大量空间；邻接表只存真实存在的边，更适合遍历。

**答案/结论**：稀疏图优先考虑邻接表或链式前向星。

### 例3：无向图建边遗漏一侧
**层次**：易错例题

**题目**：读入一条无向边 (u,v) 后，只执行 adj[u].push_back(v) 会怎样？

**分析**：这样图中只保留了单向信息，遍历和连通性结果都会出错。

**答案/结论**：无向图通常要同时加入 u→v 和 v→u。

## 练习

1. 分别写出邻接矩阵和邻接表存储同一张图的方式。
2. 思考稀疏图为什么更适合邻接表。
3. 带权无向图建边时需要保存哪些信息？
4. 尝试说明链式前向星与 vector 邻接表的共同点。

## 答案

1. 核心都是正确记录边关系，只是存法不同。
2. 因为只存存在的边，空间更省。
3. 至少要保存终点和边权，无向图还要加双向边。
4. 它们都属于“按边存邻接信息”的思路。
