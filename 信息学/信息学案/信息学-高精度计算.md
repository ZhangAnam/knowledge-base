# 信息学专题：高精度计算

> 信息学 | 数值处理 | 大整数运算

---

## 本课学什么

当题目中的整数大到超出 `long long` 范围时，就不能再依赖语言内置整数类型，而要自己模拟“按位运算”。

本课重点掌握：

- 为什么要用数组 / 字符串存大整数；
- 高精度加法、减法、乘法的进位与借位逻辑；
- 高精度题中最容易出错的细节。

---

## 核心知识 / 核心方法

### 1. 存储方式

常见做法是把一个大整数拆成若干“位”存进数组，通常低位在前，便于进位处理。

```cpp
vector<int> a;
for (int i = s.size() - 1; i >= 0; --i) {
    a.push_back(s[i] - '0');
}
```

### 2. 高精度加法

```cpp
vector<int> add(vector<int> a, vector<int> b) {
    vector<int> c;
    int t = 0;
    for (int i = 0; i < (int)max(a.size(), b.size()) || t; ++i) {
        if (i < (int)a.size()) t += a[i];
        if (i < (int)b.size()) t += b[i];
        c.push_back(t % 10);
        t /= 10;
    }
    return c;
}
```

### 3. 高精度减法

本质是逐位相减，不够减时向高位借 `1`。

前提通常是先保证被减数不小于减数。

### 4. 高精度乘低精度

```cpp
vector<int> mul(vector<int> a, int b) {
    vector<int> c;
    long long t = 0;
    for (int i = 0; i < (int)a.size() || t; ++i) {
        if (i < (int)a.size()) t += 1LL * a[i] * b;
        c.push_back(t % 10);
        t /= 10;
    }
    return c;
}
```

### 5. 常见应用

- 大整数加减乘；
- 阶乘；
- 大数斐波那契；
- 结果位数很多的计数题。

---

## 易错点 / 关键提醒

1. **前导零 / 后导零处理混乱**。
2. **低位在前还是高位在前没统一**。
3. **借位、进位逻辑写错**。
4. **输出时忘记倒序**。
5. **把高精度题误判成只需 `long long` 即可**。

## 例题

### 例1：基础例题——大整数加法

两个长度上千位的整数相加，可直接按位模拟。

### 例2：应用例题——计算 `100!`

普通整型远远不够，要不断做“高精度 × 低精度”的乘法。

---

## 练习题

1. 为什么高精度常把最低位存在数组前面？
2. 高精度减法中为什么要先比较大小？
3. 为什么计算大阶乘常用“高精度乘低精度”？

## 答案

1. 因为进位和借位都是从低位向高位处理，低位在前更方便。
2. 因为要确定结果符号并保证借位逻辑正确。
3. 因为每一步只乘一个普通整数，实现简单且效率足够。
