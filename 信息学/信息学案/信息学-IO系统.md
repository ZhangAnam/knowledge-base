# 考点6：I/O系统

> 信息学｜计算机体系结构基础｜重点理解“CPU、主存、外设如何协作”

---

## 本课学什么

I/O系统解决的是：**外部设备怎样和主机交换数据，CPU在数据传送中要参与到什么程度**。  
本课的难点是分清**程序查询、中断、DMA**三种方式的工作机制与适用场景。

学完后，你应该能够：

- 说出三种常见I/O控制方式的区别；
- 描述中断处理的基本流程；
- 判断什么时候更适合使用DMA。

---

## 核心知识

### 1. I/O系统为什么不能简单照搬CPU访问寄存器的方式

外设与CPU相比通常有以下特点：

- **速度慢**：键盘、磁盘、网络设备远慢于CPU；
- **工作节奏不稳定**：外设何时准备好数据，往往由设备本身决定；
- **种类多、接口差异大**：需要专门的控制逻辑统一管理。

因此，I/O系统的核心任务就是：  
**在“CPU很快、外设很慢”的矛盾下，让数据传送尽量高效。**

### 2. 三种常见I/O控制方式

#### 程序查询方式

CPU不断检查设备状态，直到设备就绪后再进行数据传送。

特点：

- 实现简单；
- CPU要反复轮询，空等时间多；
- 适合简单设备或数据量很小的场景。

可以把它理解成：**CPU主动问设备“你准备好了吗？”**

#### 中断方式

设备准备好后，主动向CPU发出中断请求，CPU暂停当前工作，转去处理中断。

特点：

- CPU不必一直等待；
- 比程序查询更高效；
- 适合“偶发、少量、实时性较强”的I/O。

可以把它理解成：**设备主动告诉CPU“我准备好了，你来处理我”**。

#### DMA方式

DMA控制器直接负责外设与主存之间的数据块传送，CPU只在开始和结束时参与配置或收尾。

特点：

- CPU参与程度低；
- 适合大批量数据传输；
- 能显著减少CPU在搬运数据上的时间浪费。

可以把它理解成：**CPU把搬运任务委托给DMA控制器**。

### 3. 中断处理过程

中断通常包括以下步骤：

1. **中断请求**：设备向CPU提出中断；
2. **中断响应**：CPU判断是否允许响应，并保存现场；
3. **中断处理**：转到中断服务程序执行；
4. **中断返回**：恢复现场，回到原程序继续执行。

这里最重要的理解是：

- **中断会打断当前程序，但不是丢弃当前程序**；
- CPU在处理中断前后要完成“保存现场—恢复现场”。

### 4. 中断优先级与中断屏蔽

多个设备可能同时请求中断，这时需要决定先处理谁。

- **中断优先级**：决定先响应哪个中断；
- **中断屏蔽**：在某些关键阶段暂时不响应部分中断。

常见理解：

- 高优先级设备更容易先被处理；
- 屏蔽中断是为了避免关键代码被随意打断。

### 5. DMA的工作方式

DMA传输的基本步骤通常是：

1. CPU设置传送起始地址、传送方向、传送长度；
2. DMA控制器接管总线或申请总线使用权；
3. 在外设与主存之间完成数据传送；
4. 传送结束后，用中断通知CPU。

常见DMA访问主存的方式：

| 方式 | 含义 |
| --- | --- |
| 停止CPU访问 | DMA传输期间CPU暂时不能访问主存 |
| 交替访问 | CPU和DMA轮流访问主存 |
| 周期挪用 | DMA偶尔“借用”一个存储周期完成传输 |

其中考试中最常考的是：

- **DMA不是完全不需要CPU**，它通常需要CPU先配置；
- **DMA主要减少的是CPU的数据搬运负担**，不是让CPU彻底消失。

---

## 易错点 / 关键提醒

1. **中断方式不是“设备自己把所有事做完”**  
   设备只是发请求，真正处理中断服务程序的仍然是CPU。

2. **DMA不是“外设直接和CPU交换数据”**  
   DMA强调的是**外设与主存之间**的大块数据直接传送。

3. **程序查询效率低，不是因为它不会传数据，而是因为CPU一直在等**。

4. **中断适合少量、随机到来的I/O**；  
   **DMA适合成块、大批量的I/O**。这两者的适用场景不同。

5. **中断返回前必须恢复现场**，否则原程序的执行状态会被破坏。

## 例题

### 例1：选择合适的I/O方式

某系统需要把磁盘中的一个大文件读入内存。下列方式中通常最合适的是哪一种？

A. 程序查询  
B. 中断  
C. DMA  
D. 人工操作

#### 解析

磁盘读文件属于**大批量数据传送**，如果全靠CPU逐字或逐字节搬运，效率很低。  
DMA最适合这种场景。

#### 答案

**C**

### 例2：理解中断的意义

为什么中断方式比程序查询方式更节省CPU时间？

#### 解析

程序查询要求CPU反复轮询设备状态；  
中断方式下，CPU可以先去执行别的任务，等设备准备好后再被通知。

#### 答案

**因为CPU不需要一直等待设备，而是等设备主动通知。**

### 例3：判断说法正误

“DMA方式下，CPU完全不参与I/O过程。”这句话对吗？

#### 解析

不对。CPU通常需要先对DMA进行初始化设置，传送结束后还可能接收完成中断。

#### 答案

**错误。**

### 例4：中断流程排序

请把下面步骤按正确顺序排列：  
① 中断返回 ② 中断响应并保护现场 ③ 中断请求 ④ 执行中断服务程序

#### 答案

**③ → ② → ④ → ①**

## 练习题

1. 程序查询方式为什么效率较低？  
2. 中断方式中，CPU为什么要保存现场？  
3. DMA最适合处理哪一类数据传输任务？  
4. “周期挪用”描述的是哪一种I/O相关机制？  
5. 比较中断方式和DMA方式：CPU分别参与到什么程度？

---

## 答案

1. **因为CPU要不断轮询设备状态，等待时间多**。  
2. **因为中断处理结束后还要回到原程序继续执行，必须恢复原来的运行状态**。  
3. **大批量、成块的数据传输任务**。  
4. **DMA访问主存的一种方式**。  
5. **中断方式下CPU负责具体处理；DMA方式下CPU主要负责初始化和结束处理，数据搬运主要由DMA控制器完成**。
