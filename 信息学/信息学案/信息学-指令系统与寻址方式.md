# 考点3：指令系统与寻址方式

> 信息学｜计算机体系结构基础｜重点是“指令想做什么、操作数在哪里”

---

## 本课学什么

指令系统研究的是：**CPU能执行哪些基本操作，以及一条指令如何说明操作对象的位置**。  
本课最关键的难点是理解“**操作码决定做什么，寻址方式决定去哪里找数据**”。

学完后，你应该能够：

- 说出指令的基本组成；
- 区分常见寻址方式；
- 结合简单汇编语句判断操作数到底在指令中、寄存器中还是主存中。

---

## 核心知识

### 1. 指令由什么组成

一条机器指令通常由两部分构成：

- **操作码（OP）**：说明要执行什么操作；
- **地址码**：说明操作数在哪里，或者结果送到哪里。

例如：

```asm
ADD R1, R2
```

这里 `ADD` 表示加法操作，`R1`、`R2` 对应操作对象的位置说明。

### 2. 指令系统的基本功能

从功能上看，指令大致可分为：

| 类型 | 作用 |
| --- | --- |
| 数据传送指令 | 传送数据，如 `MOV` |
| 算术逻辑指令 | 加减乘除、与或非、比较等 |
| 移位与位操作指令 | 左移、右移、按位运算 |
| 转移控制指令 | 跳转、调用、返回 |
| 输入输出指令 | 与外设交换数据 |

这说明：  
**指令系统本质上是在规定“机器能做哪些最基本动作”。**

### 3. 指令长度

常见设计有两类：

- **定长指令**：长度统一，取指和译码更规则；
- **变长指令**：短指令节省空间，长指令表达能力更强。

这也对应了两类常见架构风格：

- **RISC** 更倾向于定长、简单；
- **CISC** 更倾向于指令丰富、格式复杂。

### 4. 常见寻址方式

#### 立即寻址

操作数直接写在指令中。

```asm
MOV AX, 5
```

含义：把常数 `5` 送入寄存器。

特点：

- 取数快；
- 不必再访问主存找这个操作数；
- 但可表示的立即数范围受指令长度限制。

#### 寄存器寻址

操作数存放在寄存器中。

```asm
MOV AX, BX
```

特点：

- 速度快；
- 指令短；
- 但寄存器数量有限。

#### 直接寻址

指令中直接给出操作数所在的主存地址。

```asm
MOV AX, [1000H]
```

特点：

- 地址明确；
- 但地址本身会占用指令位数。

#### 寄存器间接寻址

寄存器中保存的是地址，真正的数据在这个地址对应的主存单元中。

```asm
MOV AX, [BX]
```

特点：

- 比直接寻址灵活；
- 适合访问数组、表格、指针类结构。

#### 相对寻址

目标地址由 **PC + 偏移量** 形成。

```asm
JMP LABEL
```

特点：

- 常用于转移指令；
- 代码位置变化时更容易重定位。

#### 基址 / 变址寻址

实际地址由“基址寄存器或变址寄存器 + 偏移量”形成。  
这类方式在访问数组、结构体、过程调用现场时很常见。

### 5. 如何判断一道题考的是什么寻址方式

可以按下面思路快速判断：

1. 数据是不是直接写在指令里？  
   - 是：立即寻址
2. 数据是不是直接在寄存器里？  
   - 是：寄存器寻址
3. 指令里写的是主存单元地址吗？  
   - 是：直接寻址
4. 寄存器里装的是地址吗？  
   - 是：寄存器间接寻址
5. 目标地址是不是 `PC + 偏移`？  
   - 是：相对寻址

### 6. CISC 与 RISC 的对比

| 对比项 | CISC | RISC |
| --- | --- | --- |
| 指令数量 | 较多 | 较少 |
| 指令格式 | 较复杂 | 较简单 |
| 指令长度 | 常见变长 | 常见定长 |
| 寻址方式 | 较丰富 | 相对精简 |
| 代表 | x86 | ARM、MIPS等 |

记忆重点：

- **CISC强调“单条指令功能强”**；
- **RISC强调“指令简单、执行规整、便于流水化”**。

---

## 易错点 / 关键提醒

1. **立即寻址中的操作数是“值”，不是“地址”**。

2. **寄存器间接寻址中，寄存器里装的是地址，不是最终数据**。

3. **相对寻址最常和转移指令联系在一起**，不要把它误看成普通数据访问。

4. **操作码和寻址方式不是一回事**  
   一个说明做什么，一个说明去哪里取数。

5. **CISC / RISC 对比是总体风格，不是绝对规则**。

## 例题

### 例1：识别寻址方式

指令：

```asm
MOV AX, 8
```

属于哪种寻址方式？

#### 解析

操作数 `8` 直接写在指令里，因此是立即寻址。

#### 答案

**立即寻址**

### 例2：再辨析一类

指令：

```asm
MOV AX, [BX]
```

这条指令为什么不是寄存器寻址？

#### 解析

因为 `BX` 中保存的是主存地址，真正的数据要到该地址所指向的内存单元里去取。

#### 答案

**因为寄存器里装的是地址，数据实际在主存中，所以属于寄存器间接寻址。**

### 例3：理解相对寻址

为什么跳转指令经常使用相对寻址？

#### 解析

相对寻址以当前PC为基准，只需保存偏移量，程序整体搬移后仍较容易保持正确跳转关系。

#### 答案

**因为它便于实现转移控制，也便于程序重定位。**

### 例4：判断说法正误

“定长指令一定比变长指令更节省存储空间。”这句话对吗？

#### 解析

不对。定长指令便于取指和译码，但不一定总是更节省空间。

#### 答案

**错误。**

## 练习题

1. 一条机器指令通常由哪两部分组成？  
2. `MOV AX, [1000H]` 属于哪种寻址方式？  
3. `MOV AX, BX` 属于哪种寻址方式？  
4. 相对寻址中的目标地址通常由什么组成？  
5. RISC为什么更容易支持规则化流水线？

---

## 答案

1. **操作码和地址码**。  
2. **直接寻址**。  
3. **寄存器寻址**。  
4. **通常由 PC 和偏移量组成**。  
5. **因为其指令更简单、格式更规整、执行步骤更统一**。
