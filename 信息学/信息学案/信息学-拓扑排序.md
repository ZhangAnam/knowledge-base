# 信息学专题：拓扑排序

> 信息学 | 图论 | 依赖关系处理

---

## 本课学什么

拓扑排序处理的是“先做谁、后做谁”的依赖关系问题。只要图中存在方向约束，且没有环，就可能转化成 DAG 上的拓扑序。

本课重点掌握：

- 什么样的图能做拓扑排序；
- 入度法是怎样一步步构造顺序的；
- 为什么“无法完成拓扑排序”往往意味着图中有环。

---

## 核心知识 / 核心方法

### 1. 拓扑排序的对象

只适用于 **有向无环图（DAG）**。

拓扑序满足：若有边 `u -> v`，则 `u` 必须排在 `v` 前面。

### 2. 入度法的核心想法

入度为 `0` 的点，说明当前没有前置依赖，可以先做。

处理它之后：

- 把它从图中“删除”；
- 相应地减少后继点的入度；
- 新出现的入度 `0` 点再入队。

### 3. Kahn 算法

```cpp
queue<int> q;
vector<int> indeg(n + 1), order;

for (int i = 1; i <= n; ++i) {
    if (indeg[i] == 0) q.push(i);
}

while (!q.empty()) {
    int u = q.front(); q.pop();
    order.push_back(u);
    for (int v : adj[u]) {
        if (--indeg[v] == 0) q.push(v);
    }
}
```

### 4. 如何判断有环

如果最后得到的拓扑序长度小于点数 `n`，就说明还有点无法被处理，也就意味着图中存在环。

### 5. 常见应用

- 课程安排；
- 任务依赖；
- 工程建造顺序；
- DAG 上 DP 的状态顺序安排。

---

## 易错点 / 关键提醒

1. **无向图也想做拓扑排序**。
2. **忘记初始化入度**。
3. **把“能排序”误当成“拓扑序唯一”**，实际上可能有多个合法顺序。
4. **队列空了就直接结束，却没检查是否处理完所有点**。

---

## 例题

### 例1：基础例题——课程安排顺序

若课程 A 是课程 B 的先修课，就连边 `A -> B`，再做拓扑排序。

### 例2：提高例题——判环

若处理结束后 `order.size() < n`，则图中有环，说明不存在合法顺序。

---

## 练习题

1. 为什么拓扑排序只适用于 DAG？
2. 入度为 `0` 的点为什么可以先处理？
3. 如何用拓扑排序判断图中是否有环？

## 答案

1. 因为有环时依赖关系会互相卡住，无法排成线性顺序。
2. 因为它没有未完成的前置依赖。
3. 若最终处理点数小于总点数，则存在环。
