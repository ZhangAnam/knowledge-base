# 信息学24：时间空间复杂度

> 信息学｜算法分析基础

## 本课学什么

学算法不能只看“能不能写出来”，还要判断“能不能在规定时间和内存内跑出来”。本课重点是学会估算时间复杂度和空间复杂度，并把它和数据规模联系起来。

## 学习目标

- 理解时间复杂度和空间复杂度的基本含义。
- 认识 O(1)、O(log n)、O(n)、O(n log n)、O(n^2) 等常见级别。
- 能根据循环结构和数据规模做初步估算。

## 概念

### 1. 时间复杂度
时间复杂度描述的是算法运行次数随输入规模变化的趋势，而不是某台电脑上的精确秒数。

### 2. 空间复杂度
空间复杂度关注算法额外使用了多少内存，包括数组、递归栈和辅助结构。

### 3. 复杂度与数据规模
复杂度分析的真正价值，在于提前判断某种做法在题目规模下是否可行。

## 算法思想 / 分析思路

- 先看循环层数和每层执行次数，再估算总操作量。
- 复杂度不能只背名字，要和 n 的大小一起看。
- 当算法递归很深或开了大数组时，也要有空间复杂度意识。

## 实现步骤

1. 读题，先看 n 的范围。
2. 分析主要循环、递归和数据结构的规模。
3. 估算时间复杂度级别，如 O(n)、O(n log n)、O(n^2)。
4. 再检查额外数组、递归深度等空间开销。
5. 将复杂度与数据规模对照，判断方案是否可行。

## 常见错误

1. **只看代码长短，不看复杂度**：代码短不代表运行快。
2. **把常数和复杂度混为一谈**：复杂度关注的是增长趋势。
3. **忽略递归栈空间**：深递归也会占用额外内存。
4. **不知道数据规模就盲目套算法**：复杂度分析必须结合 n 才有意义。

## 例题

### 例1：判断单层循环的复杂度
**层次**：基础例题

**题目**：一个从 1 到 n 的单层 for 循环，时间复杂度是多少？

**分析**：循环执行次数与 n 成正比，因此是 O(n)。

**答案/结论**：单层顺序遍历通常是 O(n)。

### 例2：为什么 O(n log n) 常优于 O(n^2)
**层次**：关键例题

**题目**：比较 n=1000 时两者的大致规模。

**分析**：n^2 约为 10^6，而 n log n 通常只有几千到几万量级，差距会越来越明显。

**答案/结论**：这也是排序中常强调 O(n log n) 的原因。

### 例3：递归深度带来的空间消耗
**层次**：易错例题

**题目**：一个递归函数每次只递归一层到 n，空间复杂度为什么不是 O(1)？

**分析**：虽然没有额外数组，但系统调用栈会保存 n 层函数信息。

**答案/结论**：递归题既要看时间，也要看栈空间。

## 练习

1. 双重循环常见的时间复杂度是什么？
2. 为什么二分查找的时间复杂度是 O(log n)？
3. 开一个长度为 n 的数组，额外空间复杂度是多少？
4. 分析算法时为什么必须结合数据规模？

## 答案

1. 若两层都与 n 同阶，通常是 O(n^2)。
2. 因为每次都把搜索区间缩小一半。
3. 通常是 O(n)。
4. 因为复杂度高低要放到具体 n 下才知道是否可行。
