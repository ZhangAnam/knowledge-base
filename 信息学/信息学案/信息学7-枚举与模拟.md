# 信息学7：枚举与模拟

> 信息学｜算法入门

## 本课学什么

很多题目没有现成公式，但只要把所有可能情况按规则试出来，就能得到正确答案。枚举强调“一个不漏”，模拟强调“按过程走”。

## 学习目标

- 理解枚举是按范围逐个尝试，模拟是按题目过程逐步执行。
- 会根据题意设计状态和循环。
- 能判断哪些题适合暴力枚举，哪些题要注意边界。

## 概念

### 1. 枚举
枚举就是把所有可能答案或所有可能状态依次尝试。它最重要的品质不是快，而是完整。

### 2. 模拟
模拟要求程序像题目描述那样一步一步执行。只要过程写得和题意一致，往往就能解决问题。

### 3. 状态
状态是当前时刻程序需要记住的信息，如位置、方向、时间、分数等。状态设计清楚，模拟才不会乱。

## 算法思想 / 分析思路

- 先确定“枚举对象”是什么，是答案、位置、时间还是方案。
- 模拟题要把自然语言过程拆成若干个可执行步骤。
- 当范围很小、规则明确时，枚举和模拟往往比复杂算法更可靠。

## 实现步骤

1. 读题，找出所有关键状态。
2. 确定枚举范围或模拟轮数。
3. 每一轮按题意更新状态。
4. 检查是否覆盖起点、终点和特殊情况。

## 常见错误

1. **漏情况**：看似写了循环，实际少考虑了某一类答案或边界。
2. **状态更新顺序错**：先改后用还是先用后改，顺序不同结果会变。
3. **把模拟写成猜公式**：题目要求按过程执行时，硬套公式往往更容易错。
4. **复杂度失控**：枚举虽然直接，但也要先估算范围是否可接受。

## 例题

### 例1：找出三位水仙花数
**层次**：基础例题

**题目**：输出所有三位水仙花数。

**分析**：三位数范围固定在 100 到 999，可以直接枚举每个数并拆位判断。

**答案/结论**：这是“小范围枚举”的标准模型。

### 例2：时钟模拟
**层次**：关键例题

**题目**：给定开始时间和经过分钟数，输出新的时刻。

**分析**：题目本质是按规则更新小时和分钟，必要时处理进位与取模。

**答案/结论**：模拟题核心是“过程忠实于题意”。

### 例3：小球反弹位置
**层次**：易错例题

**题目**：一维线上小球每秒移动一格，碰到边界就反向，问 t 秒后位置。

**分析**：除了位置，还必须记录方向；只记位置会在碰边时丢失信息。

**答案/结论**：模拟时不要漏掉方向这类隐含状态。

## 练习

1. 枚举 1 到 100 中所有能被 3 和 5 同时整除的数。
2. 模拟电梯从 1 楼依次响应若干请求后的停靠楼层。
3. 输入一个整数，判断它是否为回文数。
4. 统计 1 到 n 中个位数字为 7 的数有多少个。

## 答案

1. 条件可写成 x % 3 == 0 && x % 5 == 0。
2. 要把当前楼层作为状态逐次更新。
3. 回文数可以通过拆位逆序后比较。
4. 逐个枚举并检查个位是否为 7。
