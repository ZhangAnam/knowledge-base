# NOI学案 7：线性 DP 与区间 DP

> 信息学竞赛｜核心必修线 A5

---

## 本课学什么

本课学习两类高频 DP：线性 DP 和区间 DP。重点不是背题，而是理解“状态顺序”和“转移依赖”怎样决定 DP 能否写出来。学完后，应能先看状态之间有没有天然顺序，再决定是做线性推进还是区间合并。

## 核心模型 / 方法

### 一、线性 DP

线性 DP 适用于状态能按某个一维顺序推进的情况。

典型特征：

- `dp[i]` 只依赖前面的若干状态；
- 问题可以按下标、位置、时间等线性顺序展开。

### 二、区间 DP

区间 DP 适用于一个区间的答案由更短区间合并得到的情况。

典型特征：

- `dp[l][r]` 表示区间 `[l, r]` 的最优值；
- 枚举断点 `k`；
- 从短区间推到长区间。

### 三、状态设计的关键

先想清楚：

1. 一个状态代表什么；
2. 依赖哪些更小状态；
3. 状态间有没有天然拓扑顺序。

### 四、转移写法

- 线性 DP：常见是“当前位置由前面转来”；
- 区间 DP：常见是“当前区间由左右子区间合并”。

## 复杂度与适用条件

- 线性 DP 常见复杂度为 `O(n)` 或 `O(n^2)`。
- 区间 DP 常见复杂度为 `O(n^3)`。
- 若状态依赖不是线性的，也不是区间可合并，就要考虑其他 DP 类型。

## 易错点

1. **状态定义模糊，导致转移写不清**。
2. **区间 DP 枚举顺序错误，没有先算短区间**。
3. **线性 DP 把前缀和“最后答案”混成一个状态**。
4. **DP 只会套模板，不会解释状态意义**。

## 递进例题

### 例1：线性 DP

为什么爬楼梯、最长上升子序列等题常适合线性 DP？

**答案**：因为状态能按位置顺序推进。

### 例2：区间 DP

合并石子为什么常是区间 DP？

**答案**：因为大区间答案由左右更小区间合并而来。

### 例3：枚举顺序

区间 DP 为什么常先枚举区间长度？

**答案**：因为要保证转移时短区间结果已经算出。

### 例4：状态设计

DP 最关键的一步是什么？

**答案**：定义清楚状态含义和转移依赖。

## 练习

1. 线性 DP 的典型状态依赖有什么特点？
2. 区间 DP 常见状态写法是什么？
3. 区间 DP 为什么不能随便枚举 `l、r`？
4. DP 做不出来时应先检查什么？

## 答案

| 题号 | 答案 |
|---|---|
| 1 | 只依赖前面若干状态 |
| 2 | `dp[l][r]` |
| 3 | 因为必须保证更短区间先被计算 |
| 4 | 状态定义和转移顺序 |
