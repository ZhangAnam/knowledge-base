# NOI学案 6：背包 DP

> 信息学竞赛｜核心必修线 A5

---

## 本课学什么

本课学习背包 DP 的核心建模方式：状态怎么定义、转移为什么成立，以及 0/1、完全、多重背包之间的本质区别。学完后，应能先判断“物品能取几次”，再决定循环方向和转移方式。

## 核心模型 / 方法

### 一、背包问题的共同骨架

背包问题通常有：

- 若干物品；
- 每件物品的体积 / 费用；
- 每件物品的价值；
- 一个容量限制。

核心问题是：**在限制条件下最大化（或最小化）某个目标**。

### 二、状态设计

最基础状态：

```text
dp[j] = 容量恰为/不超过 j 时的最优值
```

### 三、0/1 背包

每件物品最多选一次。

核心转移：

```text
dp[j] = max(dp[j], dp[j-w] + v)
```

循环方向：`j` 必须 **从大到小**。

### 四、完全背包

每件物品可选无限次。

与 0/1 背包的区别不在公式，而在循环方向：`j` 通常 **从小到大**。

### 五、多重背包

每件物品有数量限制。常见处理：

- 直接枚举件数；
- 二进制拆分优化。

## 复杂度与适用条件

- 朴素背包常见为 `O(nV)`。
- 多重背包若直接枚举数量，复杂度可能更高。
- 背包 DP 的关键不是记模板，而是判断“是否可重复选”。

## 易错点

1. **0/1 背包循环方向写反**。
2. **完全背包还从大到小循环**。
3. **状态含义没想清楚，初始化乱写**。
4. **把“恰好装满”和“最多装到容量”混为一谈**。

## 递进例题

### 例1：0/1 背包

为什么 0/1 背包要从大到小枚举容量？

**答案**：为了避免同一件物品在同一轮中被重复使用。

### 例2：完全背包

为什么完全背包可以从小到大枚举容量？

**答案**：因为允许当前物品在同一轮被继续使用。

### 例3：状态理解

若题目要求“恰好装满”，初始化为什么要更谨慎？

**答案**：因为不可达状态不能随意当作 0。

### 例4：多重背包

多重背包相比 0/1 / 完全背包，多了哪个判断维度？

**答案**：每件物品可选次数有限。

## 练习

1. 0/1 背包的容量循环方向是什么？
2. 完全背包和 0/1 背包本质差别是什么？
3. 多重背包多了什么限制？
4. 背包 DP 最先应该判断什么？

## 答案

| 题号 | 答案 |
|---|---|
| 1 | 从大到小 |
| 2 | 物品是否可重复选 |
| 3 | 每件物品数量有限 |
| 4 | 状态设计和物品可选次数 |
