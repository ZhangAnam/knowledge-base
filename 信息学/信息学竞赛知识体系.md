# 信息学竞赛知识体系

> 参考 NOI 官方定位（计算机知识、算法设计、程序设计）与 IOI 官方 syllabus 的常见主题分层整理
> 目标：形成“可训练、可落地、可扩展”的竞赛知识体系，而不是只列很长的算法名词表

---

## 一、定位说明

本文件服务于 **CSP-S / NOIP / NOI / IOI 方向的竞赛训练**。

对照 NOI 与 IOI 的公开定位，竞赛训练应至少覆盖以下核心方面：

1. **算法与复杂度意识**：会分析可行性，而不是只会写代码；
2. **基础数据结构**：能组织、维护和查询数据；
3. **图与搜索**：能处理状态空间、连通性和路径问题；
4. **动态规划**：能设计状态并进行有效转移；
5. **数论、组合与字符串**：具备解决高频专题的工具箱；
6. **更高阶专题**：如计算几何、高级图论、高级数据结构等，作为进阶扩展。

因此，竞赛知识体系不能只围着“现有学案文件名”转，也不能直接把所有高级专题堆成一页长清单。更合理的方式是：

- 先明确 **核心必修线**；
- 再明确 **NOI / IOI 常见进阶线**；
- 最后保留 **扩展专题线与补学案优先级**。

## 三、NOI / IOI 常见进阶线

这部分是对照 IOI syllabus 常见主题后，属于高频但不一定在第一轮全部学完的内容。应在核心必修线稳定后继续推进。

### B1 图论进阶

- 拓扑排序
- 最小生成树
- 二分图匹配
- 强连通分量
- 网络流与最小割
- 最短路扩展：次短路、分层图、差分约束

### B2 动态规划进阶

- 树形 DP
- 状态压缩 DP
- 数位 DP
- 概率 DP
- DP 优化（单调队列优化、斜率优化、决策单调性等）

### B3 数据结构进阶

- ST 表
- 单调栈、单调队列
- 平衡树
- 可持久化数据结构
- 树链剖分
- 并查集扩展（带权并查集等）

### B4 字符串进阶

- Trie 与 AC 自动机
- 字符串 Hash 的判重与比较
- Z 函数
- 后缀数组
- 后缀自动机

### B5 数学进阶

- 逆元与同余方程
- 中国剩余定理
- 排列组合与组合恒等式
- 容斥原理
- 矩阵快速幂
- 线性递推

这部分属于更高阶的 IOI / 国家集训队训练常见主题，不要求当前目录一次性全部展开成学案，但需要在体系中明确它们的位置。

### C1 计算几何

- 向量与叉积
- 点线关系
- 凸包
- 旋转卡壳

### C2 高级图与组合结构

- 支配树、LCA、虚树等树上结构
- 高级匹配与流模型
- 更复杂的图分解与构造

### C3 高级技巧

- 分块与莫队
- CDQ 分治
- 随机化算法
- 启发式合并
- 交互题与构造题意识

这些内容应作为后续扩展，而不应挤压核心必修线。

## 五、补充拓展内容的优先顺序

若继续扩展 `信息学竞赛学案`，建议优先顺序如下：

1. `B1 图论进阶`：最小生成树、拓扑排序、网络流初步
2. `B2 动态规划进阶`：树形 DP、状态压缩 DP
3. `B4 字符串进阶`：Trie / AC 自动机
4. `B3 数据结构进阶`：单调栈、单调队列、ST 表
5. `B5 数学进阶`：组合计数、矩阵快速幂、CRT

这样能保证新增学案先补高频强依赖内容，而不是先补特别偏、特别难的专题。

## 六、建议训练顺序

### 第一阶段：建立竞赛底盘

`NOI1 → NOI2 → NOI3 → NOI4 → NOI5`

目标：

- 建立复杂度、搜索、哈希、并查集、最短路的核心模型；
- 学会“先判模型，再选算法”。

### 第二阶段：建立 DP / 数学 / 字符串能力

`NOI6 → NOI7 → NOI8 → NOI9`

目标：

- 建立状态设计、转移、模运算、字符串匹配的稳定能力。

### 第三阶段：建立树与区间结构能力

`NOI10 → NOI11`

目标：

- 完成从递归树结构到区间数据结构的过渡；
- 为后续树上问题和复杂区间维护打基础。

### 第四阶段：沿进阶线扩展

按兴趣和训练需求，在 `B1 ~ B5` 中继续推进；若面向更高阶训练，再进入 `C1 ~ C3`。

## 七、与现有目录的对应关系

1. 当前 `信息学竞赛学案` 目录优先覆盖 **核心必修线**。
2. 现有 `NOI1 ~ NOI11` 中，`NOI3` 为已补齐的桥接学案。
3. 未来若新增学案，应优先沿“图论进阶 → DP 进阶 → 字符串进阶 → 数据结构进阶 → 数学进阶”的顺序扩展。
4. 这样既能对照 NOI / IOI 大纲保持完整性，又不会让体系再次变成“看起来很全、实际落地很少”的清单。
